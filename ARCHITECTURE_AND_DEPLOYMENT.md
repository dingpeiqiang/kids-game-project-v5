## 🔄 请求流程

### 1. 用户访问网站

```
用户浏览器
    │
    ├─→ https://your-domain.com
    │       │
    │       ▼
    │   Nginx (SSL 终止)
    │       │
    │       ├─→ 静态资源 (.js, .css, .png) → 直接返回（缓存 1 年）
    │       │
    │       └─→ SPA 路由 (/index.html) → 返回前端应用
    │
    └─→ 前端应用加载完成
            │
            └─→ 发起 API 请求
```

### 2. API 请求流程

```
前端应用
    │
    ├─→ GET /api/games
    │       │
    │       ▼
    │   Nginx (反向代理)
    │       │
    │       ▼
    │   Backend (Spring Boot)
    │       │
    │       ├─→ JWT 认证
    │       ├─→ 权限校验
    │       ├─→ 查询 Redis 缓存
    │       │       │
    │       │       ├─→ Cache Hit → 返回缓存数据
    │       │       │
    │       │       └─→ Cache Miss → 查询 MySQL
    │       │               │
    │       │               ▼
    │       │           MySQL Database
    │       │               │
    │       │               └─→ 返回数据 → 写入缓存
    │       │
    │       └─→ 返回 JSON 响应
    │
    └─→ 前端渲染数据
```

### 3. 游戏资源加载

```
前端应用
    │
    ├─→ 需要加载游戏资源
    │       │
    │       ▼
    │   检查资源配置
    │       │
    │       ├─→ 使用 COS → 直接从腾讯云 CDN 加载
    │       │       │
    │       │       └─→ https://cdn.example.com/game-assets/xxx.png
    │       │
    │       └─→ 本地上传 → 从后端获取
    │               │
    │               ▼
    │           Backend API
    │               │
    │               └─→ 返回本地资源 URL
    │
    └─→ 资源加载完成，游戏启动
```

### 4. 用户登录流程

```
用户输入账号密码
    │
    ▼
前端加密发送 POST /api/auth/login
    │
    ▼
Backend 验证
    │
    ├─→ 查询 MySQL 获取用户信息
    │       │
    │       └─→ BCrypt 验证密码
    │
    ├─→ 生成 JWT Token
    │       │
    │       └─→ 包含用户 ID、角色、过期时间
    │
    └─→ 返回 Token
    
前端存储 Token (localStorage)
    │
    └─→ 后续请求携带 Token
            │
            └─→ Header: Authorization: Bearer <token>
```

## 💾 数据流向

### 数据库表关系

```
sys_user (用户表)
    │
    ├─→ sys_user_role (用户角色关联)
    │       │
    │       └─→ sys_role (角色表)
    │
    └─→ user_game_record (游戏记录)
            │
            └─→ game (游戏表)
                    │
                    └─→ game_resource (游戏资源)
                            │
                            └─→ 腾讯云 COS / 本地存储
```

### 缓存策略

```
Redis 缓存层
    │
    ├─→ user:session:{userId} → 用户会话信息 (TTL: 24h)
    ├─→ game:list → 游戏列表 (TTL: 1h)
    ├─→ game:detail:{gameId} → 游戏详情 (TTL: 30min)
    ├─→ resource:list:{gameId} → 资源列表 (TTL: 2h)
    └─→ rate:limit:{ip} → 限流计数 (TTL: 1min)
```

## 💰 成本估算（阿里云）

### 基础配置（小型部署）
- ECS: 2核 4GB - ¥200-300/月
- 带宽: 3Mbps - ¥60/月
- 磁盘: 50GB SSD - ¥25/月
- **小计**: ~¥300-400/月

### 推荐配置（中型部署）
- ECS: 4核 8GB - ¥400-500/月
- 带宽: 5Mbps - ¥100/月
- 磁盘: 100GB SSD - ¥50/月
- **小计**: ~¥550-650/月

### 额外服务
- 域名: ¥50-100/年
- SSL 证书: 免费（阿里云/Let's Encrypt）
- 腾讯云 COS: ¥10-50/月（取决于用量）

**总计**: ¥600-900/月（中型部署）

3. 升级 SSD 磁盘

```
负载均衡器 (SLB)
    │
    ├─→ ECS 1 (Frontend + Backend)
```

## 🔧 故障恢复

### 单点故障处理

| 组件 | 故障场景 | 恢复方案 | RTO | RPO |
|------|---------|---------|-----|-----|
| MySQL | 数据损坏 | 从备份恢复 | 30min | 24h |
| Redis | 缓存丢失 | 自动重建 | 5min | 0 |
| Backend | 服务崩溃 | 自动重启 | 1min | 0 |
| Frontend | 页面错误 | 回滚版本 | 5min | 0 |
| ECS | 服务器宕机 | 重新部署 | 15min | 24h |

### 备份策略
- **数据库**: 每日凌晨 2 点自动备份，保留 7 天
- **配置文件**: Git 版本控制
- **上传文件**: 腾讯云 COS（多副本）
- **日志**: 滚动保存 30 天
