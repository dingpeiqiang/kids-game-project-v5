# 流程顺序优化 - 生成资源 → 注册 → 测试

## 📋 问题描述

之前的文档中，创建新游戏的最后一步是：

```markdown
6. **注册与测试**
   - 修改 `register-game.sql`
   - 执行 SQL 注册
   - 运行 `node generate-resources.mjs`  ← ❌ 顺序错误
   - `npm run dev` 测试
```

**问题**：先注册再资源生成，导致：
- ❌ GTRS 路径错误（注册时资源还不存在）
- ❌ 前端加载失败（找不到资源配置）
- ❌ 开发体验差（需要手动重新生成）

## 🎯 优化目标

确保开发者遵循正确的顺序：**生成资源 → 注册游戏 → 启动测试**

## 💡 关键要点

### ⚠️ 必须遵守的顺序

```
生成资源 → 注册游戏 → 启动测试
   ↓           ↓          ↓
 基础        前提        验证
```

**为什么这个顺序重要？**

1. **生成资源是基础**
   - GTRS.json 必须在 `dist/` 目录
   - 图片和音频资源必须准备好
   - 否则注册信息中的路径会错误

2. **注册是前提**
   - 前端需要从数据库读取游戏配置
   - 没有注册就启动 = 找不到游戏
   - 数据库记录是运行时必需的

3. **测试是验证**
   - 验证前面所有步骤是否正确
   - 发现问题及时修正
   - 确保游戏能正常运行

### ✅ 验证清单

每次启动前，确保：
- [ ] `dist/` 目录存在且完整
- [ ] 数据库中有游戏记录
- [ ] 前端路由已配置
- [ ] package.json 已更新
