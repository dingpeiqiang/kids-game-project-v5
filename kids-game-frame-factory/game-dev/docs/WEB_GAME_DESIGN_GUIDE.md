# 🎮 网页小游戏设计指南 - 简单卡通解压风

## 🎯 核心定位

**产品类型**：网页休闲小游戏  
**目标用户**：三年级及以上儿童（6-12 岁）  
**游戏风格**：简约卡通 + 轻松解压  
**单次时长**：3-5 分钟  
**核心理念**：**简单易上手，玩完就放松**

## ❌ 不需要的设计（拒绝复杂）

### 拒绝复杂系统

```markdown
❌ 连击系统（太复杂，增加压力）
❌ 成长系统（升级/技能树，太累）
❌ 成就系统（完成任务，像上班）
❌ 排行榜（竞争压力，不放松）
❌ 经济系统（内购/货币，太现实）
❌ 社交系统（好友/公会，负担重）
❌ 每日任务（强制打卡，压力大）
❌ 赛季活动（限时完成，焦虑）
```

### 拒绝数值堆砌

```markdown
❌ 复杂的属性公式（HP = 100 * 1.05^等级）
❌ 精确的数值平衡（攻击/防御/暴击）
❌ 详细的伤害计算（物理/魔法抗性）
❌ 装备评分系统（D/C/B/A/S 评级）
❌ 战力值比较（数字攀比）
```

### 拒绝重度内容

```markdown
❌ 长篇剧情故事（没人看）
❌ 复杂的世界观（记不住）
❌ 多结局设定（太烧脑）
❌ 隐藏关卡彩蛋（找不到）
❌ 收集要素（强迫症）
```

## ✅ 需要的设计（简单就好）

### 核心原则：3 个简单

#### 1️⃣ 玩法简单

**一眼看懂**：
- ✅ 看到就知道怎么玩
- ✅ 不需要教程说明
- ✅ 操作直觉自然

**飞机大战示例**：
```markdown
✅ 简单设计：
- 拖动飞机移动
- 自动发射子弹
- 打爆敌机得分
- 撞到敌机扣血

❌ 复杂设计：
- 需要瞄准预判
- 手动切换武器
- 计算连击倍率
- 管理技能冷却
```

#### 2️⃣ 规则简单

**一句话说明**：
- ✅ "打爆敌机，别被撞到"
- ✅ "吃到食物，变长别撞墙"
- ✅ "消灭僵尸，保护房子"

**飞机大战规则**：
```markdown
✅ 简单规则：
1. 控制飞机左右移动
2. 子弹自动发射
3. 击落敌机得 10 分
4. 被撞到扣 1 滴血
5. 血量归零游戏结束

就这么简单！没有第 6 条了！
```

#### 3️⃣ 反馈简单

**即时满足**：
- ✅ 打到敌人 → 立即爆炸
- ✅ 吃到道具 → 立即生效
- ✅ 获得分数 → 立即显示
- ✅ 受到伤害 → 立即扣血

**不要延迟满足**：
- ❌ 积累能量才能放大招
- ❌ 等待冷却才能使用道具
- ❌ 结算时才给奖励

### 视觉设计：卡通可爱

#### 造型设计

**原则**：圆润可爱，颜色鲜艳

```markdown
✅ 好的设计：
- 主角：圆滚滚的小飞机
- 敌机：胖嘟嘟的小怪物
- 子弹：亮晶晶的小星星
- 道具：闪闪发光的宝石

❌ 差的设计：
- 写实风格的战斗机
- 恐怖狰狞的外星人
- 逼真的子弹炮弹
- 复杂的机械装置
```

#### 颜色搭配

**原则**：明亮鲜艳，对比强烈

```markdown
✅ 推荐配色：
- 玩家：绿色 #22c55e（清新可爱）
- 敌机：红色 #ef4444（醒目好认）
- 子弹：蓝色 #60a5fa（亮眼清晰）
- 道具：金色 #eab308（珍贵感觉）
- 背景：深蓝 #1e3a8a（突出前景）

❌ 避免配色：
- 灰暗色调（压抑）
- 低饱和度（没精神）
- 相近颜色（分不清）
- 荧光刺眼（伤眼睛）
```

#### 动画效果

**原则**：Q 弹有趣，不要太真实

```markdown
✅ 好的动画：
- idle：轻微上下浮动（像呼吸）
- 移动：平滑跟随（不瞬移）
- 受击：闪烁 + 后退（不血腥）
- 死亡：变成星星消失（不爆炸）
- 道具：旋转发光（很诱人）

❌ 差的动画：
- 真实的爆炸火光（太暴力）
- 血液飞溅效果（太吓人）
- 缓慢的过场动画（太拖沓）
- 复杂的粒子特效（太闪眼）
```

### 音频设计：轻松愉快

#### 背景音乐

**原则**：轻快愉悦，不吵闹

```markdown
✅ 好的 BGM：
- 主菜单：轻松愉快的儿歌风格
- 游戏中：节奏明快的电子音乐
- 音量：不超过音效的 50%

❌ 差的 BGM：
- 紧张刺激的战斗音乐（太压抑）
- 悲伤低沉的旋律（不开心）
- 重金属摇滚（太吵）
- 音量过大（盖过音效）
```

#### 音效设计

**原则**：清脆悦耳，有辨识度

```markdown
✅ 必需的音效：
- 射击："biu~"（短促可爱）
- 爆炸："嘭~"（闷闷的不吓人）
- 吃道具："叮铃~"（清脆好听）
- UI 点击："哒~"（轻快确认）

❌ 不需要的音效：
- 惨叫声（太吓人）
- 玻璃破碎声（太刺耳）
- 金属摩擦声（不舒服）
- 环境音（风声/雨声/鸟叫）
```

### 数值设计：凭感觉就好

#### 不要精确计算

```markdown
✅ 简单数值：
- 玩家生命：3 滴血（直观）
- 敌机生命：1 发子弹（干脆）
- BOSS 生命：10 发子弹（多打几下）
- 得分：每个敌机 10 分（整数好算）

❌ 复杂数值：
- 玩家 HP: 100 * (1.05^等级)
- 敌机 HP: 根据关卡动态调整
- 伤害公式：攻击力 - 防御力 + 随机数
- 得分系数：基础分 * 连击倍率 * 难度系数
```

#### 难度曲线：平缓舒适

```markdown
✅ 好的难度：
- 前 30 秒：几乎没有敌人（熟悉操作）
- 30 秒 -2 分钟：少量敌机（轻松应对）
- 2-3 分钟：敌机增多（稍微紧张）
- 3 分钟后：出现 BOSS（小高潮）

❌ 差的难度：
- 一开始就大量敌机（劝退）
- 难度忽高忽低（不适应）
- 完全随机生成（看运气）
- 必须完美操作才能过关（太累）
```

### 内容量：刚刚好

#### 单局时长

```markdown
✅ 理想时长：3-5 分钟
- 1 分钟：熟悉手感
- 2-3 分钟：渐入佳境
- 4-5 分钟：挑战极限
- 5 分钟后：主动结束（意犹未尽）

❌ 不理想时长：
- <1 分钟：还没玩够就结束了
- >10 分钟：太累，下次不敢玩了
```

#### 游戏模式

```markdown
✅ 只需要 1-2 种模式：
- 经典模式（主线玩法）
- 无尽模式（可选，挑战高分）

❌ 不需要多种模式：
- 生存模式/计时模式/每日挑战（太多记不住）
```

### GDD 简化模板

**网页小游戏 GDD 只要 5 页**：

```markdown
# 游戏设计文档

## 1. 一句话介绍
玩什么 + 怎么玩 + 目标是什么

例：控制飞机打敌机，躲避撞击，看谁得分高

## 2. 操作方式
- 输入方式：触摸/鼠标拖动
- 控制对象：玩家飞机
- 自动行为：自动发射子弹

## 3. 游戏对象（附参考图）

### 玩家飞机
- 外观：绿色圆形小飞机（直径 60px）
- 生命：3 滴血
- 移动速度：跟随手指/鼠标

### 敌机（1-3 种就够了）
- 小型敌机：红色圆形（直径 40px），1 发子弹击毁，得 10 分
- （可选）中型敌机：橙色圆形（直径 60px），3 发子弹，得 30 分

### 子弹
- 玩家子弹：蓝色小圆点（直径 10px）
- 飞行速度：向上快速飞行

### 道具（2-3 种就够了）
- 生命道具：红色爱心，吃了加 1 滴血
- 散射道具：黄色闪电，短时间内发射散弹

## 4. 游戏规则
1. 击落敌机得 10 分
2. 被撞到扣 1 滴血
3. 血量归零游戏结束
4. 尽量多得分数

## 5. 美术和音频风格
- 美术：卡通可爱，颜色鲜艳
- 音频：轻松愉快，音效清脆
```

## 💡 设计建议

### 从简单开始

**MVP（最小可行产品）**：
```
1 个玩家 + 1 种敌人 + 1 种子弹 + 1 种道具 = 完整游戏
```

**不要想着一口气做完**：
```
先做核心玩法 → 测试好玩 → 再考虑加内容
```

### 参考成功案例

**贪吃蛇**：
- ✅ 简单：控制方向吃食物
- ✅ 卡通：绿色小蛇 + 彩色食物
- ✅ 解压：越吃越长很有成就感

**打飞机**：
- ✅ 简单：拖动射击躲子弹
- ✅ 卡通：可爱飞机 + 萌化敌人
- ✅ 解压：击落敌机很爽快

**切水果**：
- ✅ 简单：滑动屏幕切水果
- ✅ 卡通：鲜艳水果 + 可爱效果
- ✅ 解压：一刀两断很畅快

### 记住核心

**网页小游戏的本质**：
> 让人在碎片时间放松一下，不是让人沉迷！

**三个就够了**：
- 简单的玩法
- 可爱的画面
- 轻松的体验

**其他的都是多余的负担！**
