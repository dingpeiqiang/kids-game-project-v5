## 📌 使用说明

1. **评审前准备**
   - 提前 1-2 天发出设计文档
   - 参会人员预先阅读文档
   - 准备好评审意见和问题

2. **评审会议**
   - 按本清单逐项检查
   - 记录所有评审意见
   - 给出具体的修改建议

3. **评审结论**
   - 必须给出明确的结论（通过/不通过）
   - 不通过的项目必须列出必改项
   - 所有评审人签字确认

4. **修改跟踪**
   - 设计师根据意见修改文档
   - 主评审人确认修改结果
   - 修改完成后才能进入开发
