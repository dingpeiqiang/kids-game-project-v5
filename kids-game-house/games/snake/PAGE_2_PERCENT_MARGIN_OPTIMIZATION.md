## 💡 设计原理

### 为什么选择 2%?

1. **最大化内容展示**
   - 96% 的内容区域利用率
   - 仅 4% 用于边距 (2% 上 +2% 下)
   - 视觉冲击力强

2. **适度留白**
   - 避免内容顶到屏幕边缘
   - 保持呼吸感但不过度
   - 现代扁平化设计风格

3. **自动放大**
   - 内容自动填充可用空间
   - 不需要手动调整元素大小
   - 响应式友好

4. **统一标准**
   - 三个页面使用相同边距
   - 视觉一致性高
   - 用户体验流畅

## 🔍 技术细节

### CSS 布局计算

```scss
// 容器高度计算
height: 100vh - (2% + 2%)
     = 100vh - 4%
     = 96vh

// 实际实现
.container {
  padding-top: 2%;
  padding-bottom: 2%;
  height: 96%; // 占满 96% 视口高度
}
```

### 内容自动放大

```vue
<!-- 所有子元素会自动填充可用空间 -->
<div :style="containerStyle">
  <h1 :style="titleStyle">标题</h1>  <!-- 自动放大 -->
  <button :style="buttonStyle">按钮</button>  <!-- 自动放大 -->
</div>
```

### 响应式单位优势

| 单位类型 | 优点 | 缺点 |
|---------|------|------|
| **百分比 (%)** | ✅ 自动适配<br>✅ 计算简单<br>✅ 兼容性好 | ❌ 基于父元素 |
| **视口单位 (vh/vw)** | ✅ 基于视口<br>✅ 精确控制 | ❌ 需要计算 |
| **像素 (px)** | ✅ 精确<br>✅ 直观 | ❌ 不响应式 |

**结论**: 百分比最适合此场景!
