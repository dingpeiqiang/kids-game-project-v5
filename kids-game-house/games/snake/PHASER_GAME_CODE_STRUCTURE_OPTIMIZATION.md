## 📝 最佳实践建议

### ✅ 推荐做法

1. **保持结构一致** - 所有游戏使用相同的文件结构
2. **注释清晰明确** - 每个方法都标注是否可复用
3. **配置集中管理** - 游戏特定配置放在类顶部
4. **职责单一** - 每个方法只做一件事

### ❌ 不推荐做法

1. **混合框架和游戏逻辑** - 不要在框架层写游戏特定逻辑
2. **缺少注释** - 让其他开发者难以理解哪些能复用
3. **硬编码配置** - 将配置分散在代码各处
4. **过度设计** - 为了复用而强行抽象
