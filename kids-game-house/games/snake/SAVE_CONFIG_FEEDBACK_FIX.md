## 💡 设计理念

### 为什么使用 alert？

**优点**:
1. ✅ **强制注意** - 模态对话框，用户必须点击"确定"
2. ✅ **清晰明确** - 不会错过任何信息
3. ✅ **零依赖** - 不需要额外 UI 库
4. ✅ **跨平台** - 所有浏览器都支持
5. ✅ **简单可靠** - 无兼容性问题

**替代方案对比**:

| 方案 | 优点 | 缺点 | 适用场景 |
|------|------|------|---------|
| **alert** | 强制注意、简单 | 阻塞界面 | ⭐ 重要提示 |
| **Toast** | 不阻塞、美观 | 可能被忽略 | 轻微提示 |
| **Notification** | 美观、可定制 | 需要实现 | 一般提示 |
| **状态栏** | 不干扰 | 容易被忽略 | 辅助信息 |

**选择 alert 的原因**:
- 这是**重要提示**（配置临时性）
- 需要**确保用户看到**
- 不需要复杂 UI
- 一次性提示
