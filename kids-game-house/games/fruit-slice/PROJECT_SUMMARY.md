##### 3. 炸弹生成逻辑
```typescript
private spawnFruit() {
  const config = this.LEVEL_CONFIG[this.currentLevel - 1]
  const isBomb = Math.random() < config.bombChance
  
  if (isBomb) {
    // 生成炸弹
    this.fruits.push({
      x, y, vx, vy, gravity, size, rotation, rotSpeed,
      emoji: '💣',
      color: '#333333',
      sliced: false,
      isBomb: true
    })
  } else {
    // 生成普通水果
    const fruitData = this.FRUITS[Math.floor(Math.random() * this.FRUITS.length)]
    this.fruits.push({
      x, y, vx, vy, gravity, size, rotation, rotSpeed,
      emoji: fruitData.emoji,
      color: fruitData.color,
      sliced: false,
      isBomb: false
    })
  }
}
```

##### 4. 炸弹碰撞处理

if (f.isBomb) {
  // 炸弹爆炸效果

  this.score = Math.max(0, this.score - 50)
  this.combo = 0
  this.showFloatingText('-50', f.x, f.y)

##### 5. 爆炸特效实现
```typescript
private createBombExplosion(x: number, y: number) {
  // 40个彩色粒子
  for (let i = 0; i < 40; i++) {
    const angle = Math.random() * Math.PI * 2
    const speed = 6 + Math.random() * 12
    
    this.particles.push({
      x, y,
      vx: Math.cos(angle) * speed,
      vy: Math.sin(angle) * speed - 5,
      life: 1,
      color: i % 3 === 0 ? '#FF4444' : 
             i % 3 === 1 ? '#FFAA00' : '#333333',
      size: 4 + Math.random() * 8,
      type: 'juice'
    })
  }
  
  // 强烈屏幕震动
  this.screenShake = 15
}
```

## 🎯 关卡设计亮点

### 三阶段难度曲线

#### 🟢 阶段1：新手期（关卡1-3）
- **炸弹概率**: 0% → 10%
- **水果数量**: 2-3个
- **速度倍率**: 1.0x - 1.4x
- **设计目标**: 让玩家熟悉基本操作，建立信心

#### 🟡 阶段2：进阶期（关卡4-7）
- **炸弹概率**: 15% → 30%
- **水果数量**: 4-5个
- **速度倍率**: 1.6x - 2.2x
- **设计目标**: 考验判断力和反应速度，需要策略

#### 🔴 阶段3：高手期（关卡8-10）
- **炸弹概率**: 35% → 45%
- **水果数量**: 6-8个
- **速度倍率**: 2.5x - 3.0x
- **设计目标**: 极限挑战，测试玩家的最高水平

### 平衡性设计原则

1. **渐进式提升**: 每关小幅增加难度，避免突变
2. **多维度挑战**: 同时提升速度、数量、炸弹概率
3. **风险收益**: 炸弹威胁 vs 高分奖励的平衡
4. **容错空间**: 惩罚严厉但不过度，保持可玩性

## 🎓 技术要点

### 关键技术决策

1. **炸弹标识方式**
   - 选择: 在Fruit接口中添加`isBomb`布尔字段
   - 原因: 简单直观，易于扩展

2. **概率控制方式**
   - 选择: 每个关卡配置独立的`bombChance`
   - 原因: 灵活可控，便于调整平衡

3. **惩罚机制设计**
   - 选择: 扣50分 + 重置连击
   - 原因: 严厉但不过度，保持游戏可玩性

4. **视觉效果实现**
   - 选择: 40个粒子 + 强烈震动
   - 原因: 明显的反馈，印象深刻

### 性能优化技巧

1. **对象复用**: 粒子和水果及时清理
2. **高效渲染**: Canvas API + requestAnimationFrame
3. **碰撞优化**: 距离检测算法优化
4. **内存管理**: 及时移除无用对象
