# 道具视觉效果增强

## 问题描述

用户反馈："画面中的可用道具显示状态不够明显，不太容易与敌人区分"

**问题分析：**
- 道具只有一个简单的emoji图标
- 金色光晕较弱（15px模糊）
- 没有明显的边框或背景
- 在混乱的战斗场景中容易被忽略
- 与敌人颜色相近时难以区分

---

## 优化方案

### 1. 添加脉冲动画

**新增代码：**
```typescript
// 脉冲动画（让道具更醒目）
const pulse = 1 + Math.sin(Date.now() / 150) * 0.15
ctx.scale(pulse, pulse)
```

**效果：**
- 道具大小在 **85% - 115%** 之间脉动
- 周期约 **0.94秒**（150ms × 2π）
- 吸引玩家注意力
- 与静态的敌人形成对比

---

### 2. 三层视觉结构

#### 外层：强光晕

**修改前：**
```typescript
ctx.fillStyle = 'rgba(255,215,0,0.3)'
ctx.shadowBlur = 15
ctx.beginPath(); ctx.arc(0, 0, 14, 0, Math.PI * 2); ctx.fill()
```

**修改后：**
```typescript
ctx.fillStyle = 'rgba(255,215,0,0.4)'  // ✅ 透明度+33%
ctx.shadowBlur = 25                    // ✅ 光晕+67%
ctx.beginPath(); ctx.arc(0, 0, 18, 0, Math.PI * 2); ctx.fill()  // ✅ 半径+29%
```

**改进：**
- 光晕范围从14px增加到**18px**
- 模糊强度从15px增加到**25px**
- 透明度从30%提升到**40%**
- 形成明显的"光环"效果

---

#### 中层：白色边框

**新增代码：**
```typescript
// 中层圆环（白色边框）
ctx.strokeStyle = '#FFFFFF'
ctx.lineWidth = 2
ctx.shadowBlur = 10
ctx.beginPath(); ctx.arc(0, 0, 14, 0, Math.PI * 2); ctx.stroke()
```

**效果：**
- 清晰的白色圆环边框
- 与任何背景色都形成对比
- 明确标识道具边界
- 发光效果增强可见性

---

#### 内层：深色背景

**新增代码：**
```typescript
// 内层背景（深色，突出图标）
ctx.fillStyle = 'rgba(0,0,0,0.6)'
ctx.beginPath(); ctx.arc(0, 0, 12, 0, Math.PI * 2); ctx.fill()
```

**效果：**
- 半透明黑色背景
- 为emoji图标提供对比底色
- 确保图标在任何背景下都清晰可见
- 直径24px的圆形区域

---

### 3. 图标增强

**修改前：**
```typescript
ctx.font = '16px sans-serif'
ctx.fillText(puIcons[p.type] || '?', 0, 0)
```

**修改后：**
```typescript
ctx.font = 'bold 20px sans-serif'  // ✅ 粗体 + 尺寸+25%
ctx.fillStyle = '#FFFFFF'           // ✅ 纯白色
ctx.fillText(puIcons[p.type] || '?', 0, 1)  // ✅ 微调位置
```

**改进：**
- 字体从16px增加到**20px**（+25%）
- 添加**粗体**样式
- 使用纯白色，最大化对比度
- 垂直位置微调1px，视觉居中

## 性能考虑

### 渲染开销

**增加的绘制操作：**
- 原来：1次arc + 1次fill + 1次fillText = **3次操作**
- 现在：3次arc + 2次fill + 1次stroke + 1次fillText + 1次scale = **8次操作**

**影响评估：**
- ✅ 道具数量有限（最多10-15个）
- ✅ Canvas 2D渲染效率高
- ✅ 脉冲动画只需计算sin值
- ⚠️ 每帧增加约5次绘制操作

**总开销：**
- 假设10个道具：10 × 5 = 50次额外操作/帧
- 60 FPS下：50 × 60 = 3000次额外操作/秒
- 现代浏览器完全可以承受
