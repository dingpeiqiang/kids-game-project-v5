# 📱 手机端适配优化 v2 - 进一步缩小

## 🎯 优化目标

用户反馈"还是太大"，需要进一步缩小游戏元素和攻击范围。

---

## ✅ 优化内容

### 1. Canvas尺寸进一步缩小

**修改前**：
- 手机端：屏幕宽度的90%，高度的70%
- 桌面端：屏幕宽度的90%，高度的70%

**修改后**：
- **手机端**：屏幕宽度的**85%**，高度的**60%** ⬇️
- **桌面端**：屏幕宽度的90%，高度的70%（保持不变）

**代码位置**: `config.ts`

```typescript
const isMobile = windowWidth < 768
const widthPercent = isMobile ? 0.85 : 0.9  // 手机端85%，桌面端90%
const heightPercent = isMobile ? 0.6 : 0.7   // 手机端60%，桌面端70%
```

---

### 2. 攻击范围进一步缩小

**修改前**：
- 基础射程：200像素
- iPhone: 200 × 0.94 = 188px
- Android: 200 × 0.90 = 180px

**修改后**：
- **基础射程：150像素** ⬇️ 减少25%
- iPhone: 150 × 0.94 = **141px** ⬇️ 减少25%
- Android: 150 × 0.90 = **135px** ⬇️ 减少25%

**代码位置**: `combat.ts`

```typescript
// playerShoot函数
const attackRange = 150 * SCALE_RATIO  // 从200改为150

// drawPlayer函数
const attackRange = 150 * SCALE_RATIO  // 从200改为150
```

## 🔧 技术实现

### 设备检测逻辑

```typescript
const isMobile = windowWidth < 768
```

**判断标准**：
- 宽度 < 768px：判定为手机/小平板
- 宽度 ≥ 768px：判定为桌面/大平板

---

### 响应式配置

```typescript
// 手机端配置
if (isMobile) {
  widthPercent = 0.85   // 85% 屏幕宽度
  heightPercent = 0.6   // 60% 屏幕高度
} else {
  // 桌面端配置
  widthPercent = 0.9    // 90% 屏幕宽度
  heightPercent = 0.7   // 70% 屏幕高度
}
```
