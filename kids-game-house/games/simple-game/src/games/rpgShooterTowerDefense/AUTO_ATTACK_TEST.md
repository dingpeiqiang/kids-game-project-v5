# 🎯 玩家自动攻击 - 快速测试指南

## 🚀 测试步骤

### 1. 启动游戏
```
访问: http://localhost:5102
点击: "RPG塔防射击"
操作: 点击屏幕或按空格键开始
```

---

### 2. 观察自动攻击

**预期现象**：
- ✅ 敌人出现后，玩家自动开始射击
- ✅ 子弹朝向最近的敌人飞行
- ✅ 每次射击都有"咻"的音效
- ✅ 不需要任何操作，完全自动

---

### 3. 检查视觉指示器

**应该看到**：

#### 🔵 攻击范围圈
- 位置：以玩家为中心的圆圈
- 颜色：淡蓝色半透明
- 样式：虚线
- 半径：300像素

#### 🔴 目标标记
- 位置：当前锁定的敌人身上
- 颜色：红色
- 形状：圆圈（半径15像素）
- 效果：跟随敌人移动

#### ➡️ 瞄准线
- 起点：玩家中心
- 终点：目标敌人中心
- 颜色：红色半透明
- 宽度：2像素

---

### 4. 测试多目标场景

**操作步骤**：
1. 等待多个敌人出现在屏幕上
2. 观察玩家选择哪个敌人
3. 移动玩家靠近不同的敌人
4. 观察目标是否切换

**预期结果**：
- ✅ 始终攻击最近的敌人
- ✅ 当更近的敌人出现时，立即切换目标
- ✅ 切换过程平滑，无闪烁

---

### 5. 测试射程边界

**操作步骤**：
1. 站在远处观察敌人（距离>300像素）
2. 确认玩家不射击
3. 慢慢向敌人移动
4. 观察何时开始射击

**预期结果**：
- ✅ 距离>300：不射击，安静
- ✅ 距离≤300：立即开始射击
- ✅ 蓝色虚线圆圈清晰显示边界

---

### 6. 测试建造模式

**操作步骤**：
1. 按B键进入建造模式
2. 选择炮台并放置
3. 观察玩家是否仍在射击

**预期结果**：
- ✅ 建造模式下玩家继续射击
- ✅ 不影响炮台放置
- ✅ 两者独立运行

---

### 7. 听音效反馈

**应该听到**：
- 🔊 每次射击：短促的"咻"声（400Hz）
- 🔊 频率：约每秒4次（250ms间隔）
- 🔊 音量：适中，不刺耳

**如果没有声音**：
- 检查浏览器是否允许音频
- Chrome/Safari需要先与页面交互
- 尝试点击页面任意位置后再试

---

## 🐛 常见问题排查

### 问题1：玩家不射击

**可能原因**：
- ❌ 没有敌人在射程内
- ❌ 游戏未开始
- ❌ 浏览器阻止了JavaScript

**解决方法**：
1. 确认敌人已进入蓝色圆圈内
2. 确认已点击屏幕开始游戏
3. 检查控制台是否有错误

---

### 问题2：看不到瞄准线

**可能原因**：
- ❌ 没有敌人在射程内
- ❌ Canvas渲染问题

**解决方法**：
1. 确保有敌人在300像素范围内
2. 刷新页面重试
3. 检查浏览器兼容性

---

### 问题3：听不到音效

**可能原因**：
- ❌ 浏览器阻止自动播放
- ❌ 系统音量为0
- ❌ 浏览器不支持Web Audio API

**解决方法**：
1. 点击页面任意位置（激活音频上下文）
2. 检查系统音量
3. 尝试Chrome/Edge/Firefox浏览器

---

### 问题4：目标选择不正确

**可能原因**：
- ❌ 算法bug
- ❌ 距离计算错误

**调试方法**：
打开浏览器控制台（F12），运行：
```javascript
// 查看玩家位置
console.log('Player:', gameStateRef.current.player)

// 查看所有敌人
console.log('Enemies:', gameStateRef.current.enemies)

// 手动计算距离
const dx = enemy.x - player.x
const dy = enemy.y - player.y
const dist = Math.sqrt(dx*dx + dy*dy)
console.log('Distance:', dist)
```

## 📊 性能监控

### 检查FPS
在浏览器控制台中运行：
```javascript
// 简单FPS计数器
let frames = 0
let lastTime = performance.now()

function countFPS() {
  frames++
  const now = performance.now()
  if (now - lastTime >= 1000) {
    console.log('FPS:', frames)
    frames = 0
    lastTime = now
  }
  requestAnimationFrame(countFPS)
}

countFPS()
```

**预期结果**：稳定在60 FPS

---

### 检查内存使用
在Chrome DevTools中：
1. 打开Performance标签
2. 录制10秒
3. 观察Memory曲线
4. 应该平稳，无持续增长
