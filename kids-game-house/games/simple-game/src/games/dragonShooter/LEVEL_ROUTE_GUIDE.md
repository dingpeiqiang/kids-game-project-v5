## 🎨 方法一：可视化编辑器（推荐）

### ✅ 优点
- 无需编程知识
- 所见即所得
- 实时预览效果
- 永久保存到浏览器

### 📝 使用步骤

#### 1. 进入编辑模式
```
游戏开始界面 → 点击 "✏️ 绘制路线" 按钮
```

#### 2. 绘制路线
- **鼠标/触摸拖动**：在画布上自由绘制
- **起点**：绿色圆点 🟢（龙出现的位置）
- **终点**：红色圆点 🔴（龙消失的位置）
- **提示**：可以从画面外开始，让龙飞入

#### 3. 保存路线
```
绘制完成 → 点击 "💾 保存" → 输入名称 → 确定
```

**命名建议：**
- `第1关 - 简单入门`
- `第5关 - 螺旋挑战`
- `BOSS关 - 终极战场`

#### 4. 在游戏中使用
保存的路线会自动在游戏中随机分配给龙。

### 示例2：创建自定义螺旋路线

如果你想自定义螺旋的参数，可以修改 `generateLevelRoute` 函数：

```typescript
case 'spiral':
  for (let i = 0; i <= totalPoints; i++) {
    const progress = i / totalPoints
    // 修改这里的参数来调整螺旋效果
    const radius = 80 + progress * 40  // 更大的半径
    const rotations = 12  // 更多的旋转圈数
    points.push({
      x: CENTER_X + Math.cos(progress * Math.PI * rotations) * radius,
      y: -200 + progress * (BASE_H + 400)
    })
  }
  break
```

### 示例3：完全自定义路线

如果你想完全手动定义路线点：

```typescript
7: {
  id: 'level_7_manual',
  name: '第7关 - 手工路线',
  points: [
    { x: 180, y: -200 },  // 起点（画面外上方）
    { x: 180, y: 0 },     // 进入画面
    { x: 250, y: 100 },   // 向右移动
    { x: 100, y: 200 },   // 向左移动
    { x: 180, y: 300 },   // 回到中间
    { x: 180, y: 640 },   // 到达底部
    { x: 180, y: 840 }    // 终点（画面外下方）

```

**注意：** 手动定义的路线需要至少1600个点才能确保龙完整显示，建议使用程序生成。

## 📐 路线设计最佳实践

### ✅ 推荐做法

1. **起点在画面外**
   ```typescript
   y: -200  // 从上方200像素外开始
   ```

2. **终点在画面外**
   ```typescript
   y: BASE_H + 400  // 到底部下方400像素
   ```

3. **保持1600个点**
   ```typescript
   const totalPoints = 1600  // 确保足够长
   ```

4. **X轴范围合理**
   ```typescript
   x: CENTER_X ± 80  // 不要超出屏幕太多
   ```

### ❌ 避免的做法

1. **路线太短**
   - 少于500点会导致龙很快消失
   
2. **X轴偏移过大**
   - 超过150像素会让龙飞出屏幕

3. **Y轴范围不足**
   - 必须覆盖 `-200` 到 `BASE_H + 400`

## 📞 常见问题

### Q1: 为什么龙不按照我设计的路线走？
**A:** 检查以下几点：
1. 关卡号是否正确匹配
2. 路线是否有至少1600个点
3. Y轴范围是否覆盖 `-200` 到 `BASE_H + 400`

### Q2: 如何让路线更复杂？
**A:** 可以叠加多个波形：
```typescript
const wave1 = Math.sin(progress * Math.PI * 6) * 50
const wave2 = Math.cos(progress * Math.PI * 10) * 30
x: CENTER_X + wave1 + wave2
```

### Q3: 可以分享我的路线配置吗？
**A:** 可以！复制 `LEVEL_SPECIFIC_ROUTES` 的配置分享给其他人，他们粘贴到自己的代码中即可使用。

### Q4: 如何让不同关卡使用不同的速度？
**A:** 修改 `getSpeedMultiplier` 函数（约570行）：
```typescript
function getSpeedMultiplier(level: number): number {
  if (level === 5) return 1.3  // 第5关更快
  if (level === 10) return 1.5 // 第10关非常快
  return 1.0  // 默认速度
}
```
