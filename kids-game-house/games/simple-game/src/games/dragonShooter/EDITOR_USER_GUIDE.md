## 功能概述

路线编辑器是 DragonShooter 游戏中用于自定义龙移动路径的工具。通过可视化绘制，您可以创建独特的游戏路线，让龙按照您设计的路径移动。

### 核心功能

✅ **可视化绘制** - 鼠标/触摸直接绘制路线  
✅ **智能优化** - 自动抽稀和圆滑处理  
✅ **多路线支持** - 可以创建多条路线  
✅ **实时预览** - 绘制时即时显示  
✅ **数据导出** - 支持导出为 JSON 文件  
✅ **持久化保存** - 保存到本地存储

## 快速开始

### 第一步：进入编辑模式

1. 启动游戏
2. 在主菜单点击 **"绘制路线"** 按钮
3. 进入路线编辑界面

### 第二步：绘制第一条路线

1. **在虚线框内**按下鼠标左键（或触摸屏幕）
2. 拖动鼠标绘制您想要的路线
3. 松开鼠标完成绘制

> 💡 **提示**：路线应该从屏幕顶部开始，到底部结束

### 第三步：保存路线

1. 点击底部的 **"💾 保存"** 按钮
2. 看到提示 "✅ 已保存 X 条路线！路线仍在编辑器中"
3. 路线已保存到本地存储

### 第四步：测试路线

1. 点击 **"⬅️ 返回"** 回到主菜单
2. 点击 **"开始闯关"**
3. 观察龙是否沿着您绘制的路线移动

## 详细操作说明

### 🎨 绘制路线

#### 基本绘制

```
1. 在虚线框区域内按下鼠标/触摸
   ↓
2. 保持按下状态，移动鼠标/手指
   ↓
3. 路线会跟随您的移动实时显示
   ↓
4. 松开鼠标/手指，完成绘制
```

#### 继续绘制

如果您想在现有路线上添加更多点：

```
1. 确保当前有路线（至少3个点）
   ↓
2. 在路线末尾附近再次按下并拖动
   ↓
3. 新点会自动添加到路线末尾
```

#### 绘制新路线

如果要创建完全不同的路线：

```
1. 点击 "➕ 新建" 按钮
   ↓
2. 开始绘制新的路线
   ↓
3. 旧路线仍然保留，可以同时存在多条路线
```

### ⚠️ 注意事项

1. **必须在虚线框内绘制**
   - 虚线框外点击不会生效
   - 虚线框范围：360x640 像素

2. **至少需要3个点**
   - 少于3个点的路线无法保存
   - 系统会提示 "⚠️ 每条路线至少3个点!"

3. **避免过于密集的點**
   - 点与点之间距离太近会被自动过滤
   - 建议保持适当间距，使路线更流畅

## 按钮功能详解

底部有6个功能按钮，从左到右依次是：

### 1. ➕ 新建（紫色）

**功能**：创建一条新的空路线

**使用场景**：
- 想绘制第二条、第三条路线时
- 想重新开始绘制时

**效果**：
- 创建新的空路线
- 原有路线保留
- 显示提示："➕ 已新建路线，请开始绘制"

**示例**：
```
已有路线A → 点击"新建" → 可以绘制路线B
结果：路线A和路线B都存在
```

---

### 2. 🗑️ 清除（红色）

**功能**：清除所有路线

**使用场景**：
- 想完全重新开始
- 绘制的路线都不满意

**效果**：
- 删除所有已绘制的路线
- 编辑器变为空白
- 显示提示："🗑️ 已清除"

**⚠️ 警告**：此操作不可撤销！

---

### 3. 💾 保存（绿色）

**功能**：保存所有路线到本地存储

**使用场景**：
- 绘制完成后保存
- 定期保存以防丢失

**效果**：
- 将所有路线保存到 localStorage
- 自动进行优化处理（抽稀+圆滑）
- 显示提示："✅ 已保存 X 条路线！路线仍在编辑器中"
- **路线不会被清空**，可以继续编辑

**技术细节**：
- 保存格式：JSON
- 存储位置：浏览器 localStorage
- 键名：`dragonShooter_customRoutes`
- 坐标系统：游戏坐标（已减去偏移）

---

### 4. ✨ 优化（橙色）

**功能**：优化当前路线（抽稀+圆滑）

**使用场景**：
- 绘制的路线点太多，需要简化
- 路线不够平滑，需要圆滑处理

**效果**：
- 减少不必要的点（Douglas-Peucker 算法）
- 使曲线更平滑（Catmull-Rom 插值）
- 显示点数变化："✨ 100→30点 圆滑完成"

**优化前后对比**：
```
优化前：100个点，锯齿状
   ↓
优化后：30个点，平滑曲线
```

**建议**：
- 先绘制粗糙的路线
- 然后点击"优化"获得平滑效果
- 最后再"保存"

---

### 5. 📥 导出（蓝色）

**功能**：导出路线为 JSON 文件

**使用场景**：
- 备份路线数据
- 分享给其他人
- 在其他设备上使用

**效果**：
- 生成 JSON 文件
- 自动下载文件
- 文件名格式：`routes_时间戳.json`
- 显示提示："✅ 已导出 X 条路线!"

**JSON 文件格式**：
```json
{
  "version": "1.0",
  "lastModified": "2026-05-03T12:00:00.000Z",
  "routes": [
    {
      "id": "route_1234567890_0",
      "name": "路线1",
      "points": [
        {"x": 180, "y": 0},
        {"x": 185, "y": 50},
        {"x": 190, "y": 100}
      ]
    }
  ]
}
```

---

### 6. ⬅️ 返回（主题色）

**功能**：返回主菜单

**使用场景**：
- 完成编辑，准备开始游戏
- 想退出编辑模式

**效果**：
- 返回游戏主菜单
- 已保存的路线会保留
- 未保存的路线会丢失（⚠️ 注意先保存）

## 常见问题

### Q1: 为什么我画的路线保存后不见了？

**A**: 可能的原因：
1. ❌ 没有点击"保存"按钮就直接返回了
2. ❌ 路线点数少于3个
3. ❌ 点击了"清除"按钮

**解决方案**：
- ✅ 绘制完成后务必点击"💾 保存"
- ✅ 确保路线至少有3个点
- ✅ 看到绿色提示后再返回

---

### Q2: 如何绘制多条路线？

**A**: 步骤如下：

```
1. 绘制第一条路线
   ↓
2. 点击"💾 保存"
   ↓
3. 点击"➕ 新建"
   ↓
4. 绘制第二条路线
   ↓
5. 再次点击"💾 保存"
   ↓
结果：两条路线都保存了
```

---

### Q3: 路线画错了怎么办？

**A**: 有三种方法：

**方法1：重新绘制整条路线**
```
1. 点击"🗑️ 清除"（会删除所有路线）
2. 重新绘制
```

**方法2：只删除当前路线**
```
目前不支持单条路线删除
建议：清除后重新绘制所有路线
```

**方法3：优化修正**
```
1. 如果路线点太多或不平滑
2. 点击"✨ 优化"
3. 系统会自动修正
```

---

### Q4: 保存的路线在哪里？

**A**: 保存在浏览器的 localStorage 中

**查看方法**：
1. 打开浏览器开发者工具（F12）
2. 切换到 Application/Storage 标签
3. 展开 Local Storage
4. 找到 `dragonShooter_customRoutes`

**备份方法**：
- 使用"📥 导出"功能导出为 JSON 文件

---

### Q5: 如何在不同设备间同步路线？

**A**: 使用导出/导入功能

**步骤**：
```
设备A：
1. 点击"📥 导出"
2. 保存 JSON 文件

设备B：
1. 暂时不支持直接导入
2. 需要手动将 JSON 文件内容
   复制到代码中的 routes 数组
```

> 💡 **未来计划**：添加导入功能

---

### Q6: 为什么龙不按照我画的路线走？

**A**: 可能的原因：

1. **坐标偏移问题**（已修复）
   - 确保使用的是最新版本
   - 重新绘制并保存路线

2. **路线起点不在顶部**
   - 路线应该从 y=0 附近开始
   - 到 y=640 附近结束

3. **路线点太少**
   - 至少需要3个点
   - 建议10-50个点

**检查方法**：
```
1. 进入编辑模式
2. 查看已保存的路线
3. 确认路线形状正确
4. 如有问题，重新绘制
```

---

### Q7: 可以点击"新建"来清空当前路线吗？

**A**: **不可以！**

- "➕ 新建"是创建**新路线**，不是清空
- 原有路线会保留
- 如果想清空，应该用"🗑️ 清除"

**正确做法**：
```
想清空所有路线 → 点击"🗑️ 清除"
想添加新路线   → 点击"➕ 新建"
```

## 最佳实践

### 🎯 绘制技巧

#### 1. 规划先行

在绘制前先想好路线的大致形状：
- 直线型：适合简单关卡
- 波浪型：增加趣味性
- S型：挑战性更高
- 螺旋型：高难度关卡

#### 2. 控制点数

- **推荐**：20-50个点
- **最少**：3个点
- **最多**：建议不超过200个点（会影响性能）

#### 3. 保持间距

- 点与点之间保持 10-30 像素间距
- 避免过于密集的点
- 转弯处可以适当加密

#### 4. 善用优化

```
绘制流程：
1. 快速绘制粗糙路线（不用太精确）
2. 点击"✨ 优化"自动圆滑
3. 检查效果，不满意则重新绘制
4. 满意后点击"💾 保存"
```

---

### 📐 路线设计规范

#### 起点和终点

```
起点：y ≈ 0（屏幕顶部）
终点：y ≈ 640（屏幕底部）
X坐标：建议在 50-310 范围内
```

#### 路线长度

- **短路线**：50-100个点，适合简单关卡
- **中路线**：100-200个点，适合中等关卡
- **长路线**：200-400个点，适合高难度关卡

#### 路线形状

✅ **推荐的形状**：
- 平缓的波浪
-  gentle 曲线
- 适度的转折

❌ **避免的形状**：
- 急转弯（龙可能跟不上）
- 重叠交叉（视觉效果差）
- 超出边界（龙会消失）

---

### 💾 保存策略

#### 定期保存

```
建议保存时机：
1. 每绘制完一条路线后
2. 点击"优化"后
3. 准备退出编辑器前
```

#### 备份重要路线

```
对于精心设计的路线：
1. 点击"💾 保存"（保存到 localStorage）
2. 点击"📥 导出"（备份为 JSON 文件）
3. 将 JSON 文件保存到安全位置
```

#### 版本管理

```
如果需要多个版本：
1. 绘制版本1 → 导出为 routes_v1.json
2. 修改路线   → 导出为 routes_v2.json
3. 根据需要选择使用哪个版本
```

## 技术细节

### 坐标系统

#### 游戏坐标 vs 画布坐标

```
画布大小：640 x 800 像素
游戏区域：360 x 640 像素
偏移量：X=140, Y=100

转换公式：
画布坐标 = 游戏坐标 + 偏移量
游戏坐标 = 画布坐标 - 偏移量
```

#### 坐标范围

```
游戏坐标系：
X: 0 ~ 360
Y: 0 ~ 640

推荐绘制范围：
X: 50 ~ 310（留边距）
Y: 0 ~ 640（全高度）
```

---

### 优化算法

#### Douglas-Peucker 抽稀

**作用**：减少不必要的点

**原理**：
```
1. 连接起点和终点
2. 找出距离线段最远的点
3. 如果距离 > 阈值，保留该点
4. 递归处理左右两段
```

**效果**：
- 100点 → 30点（减少70%）
- 保持路线形状不变
- 提高渲染性能

---

#### Catmull-Rom 圆滑

**作用**：使曲线更平滑

**原理**：
```
1. 使用弦长参数化
2. 三次样条插值
3. 自动生成中间点
4. 消除尖锐拐角
```

**效果**：
- 锯齿状 → 平滑曲线
- 自然的转弯
- 更好的视觉效果

### 性能考虑

#### 点数限制

- **最小**：3个点（否则无法保存）
- **推荐**：20-50个点
- **最大**：建议不超过500个点

#### 渲染优化

- 使用对象池减少GC
- 缓存计算结果
- 按需渲染

#### 内存占用

- 每条路线约 1-5 KB
- 10条路线约 10-50 KB
- localStorage 限制通常 5-10 MB

## 故障排除

### 问题1：点击按钮没反应

**可能原因**：
- 浏览器兼容性问题
- JavaScript 错误

**解决方法**：
1. 刷新页面重试
2. 检查浏览器控制台是否有错误
3. 尝试其他浏览器

---

### 问题2：路线显示不正常

**可能原因**：
- 坐标系统问题
- 数据损坏

**解决方法**：
1. 清除所有路线重新绘制
2. 检查是否在虚线框内绘制
3. 确保使用最新版本

---

### 问题3：保存失败

**可能原因**：
- localStorage 已满
- 浏览器限制了存储

**解决方法**：
1. 清理浏览器缓存
2. 导出为 JSON 文件备份
3. 减少路线数量
