## 🔍 调试方法

### 1. 检查加载状态

在浏览器控制台输入：

```javascript
// 检查是否已加载
console.log('加载状态:', routeLoader.isLoaded())

// 查看可用关卡
console.log('可用关卡:', routeLoader.getAvailableLevels())

// 查看统计信息
console.log('统计:', routeLoader.getStats())

// 获取特定关卡路线
const route = routeLoader.getLevelRoute(1)
console.log('第1关路线:', route.name, route.points.length, '个点')
```

### 2. 检查网络请求

打开浏览器的 Network 标签，刷新页面，应该看到：
- `GET /games/dragonShooter/routes/level_routes.json` - 200 OK
- `GET /games/dragonShooter/routes/custom_routes.json` - 200 OK

如果看到 404 错误，说明文件路径不正确。

### 3. 查看错误信息

如果加载失败，控制台会显示：
```
❌ 加载路线失败: [错误信息]
⚠️  关卡路线文件不存在，使用默认路线
```

## 🐛 常见问题排查

### 问题1：路线没有加载

**症状：** 控制台显示 "⚠️ 关卡路线文件不存在"

**原因：** JSON 文件路径不正确或文件不存在

**解决：**
1. 确认文件存在：
   ```bash
   ls routes/level_routes.json
   ls routes/custom_routes.json
   ```

2. 检查 Vite 配置，确保允许访问 routes 目录

3. 尝试直接访问文件：
   ```
   http://localhost:5173/games/dragonShooter/routes/level_routes.json
   ```

### 问题2：JSON 格式错误

**症状：** 控制台显示 "❌ 加载路线失败: SyntaxError"

**原因：** JSON 文件格式不正确

**解决：**
1. 使用 JSON 验证工具检查文件
2. 重新运行生成脚本：
   ```bash
   npx ts-node generateInitialRoutes.ts
   ```

### 问题3：路线不生效

**症状：** 龙仍然使用旧的路线

**原因：** 浏览器缓存

**解决：**
1. 硬刷新：Ctrl + Shift + R (Windows) 或 Cmd + Shift + R (Mac)
2. 或者清除浏览器缓存
3. 或者在 URL 后添加时间戳参数

### 问题4：TypeScript 编译错误

**症状：** 编译时出现类型错误

**解决：**
1. 确保 `routeLoader.ts` 导出了正确的类型
2. 检查 `CustomRoute` 类型定义
3. 重启 TypeScript 服务器
