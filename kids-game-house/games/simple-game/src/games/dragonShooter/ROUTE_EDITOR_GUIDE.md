# 路线编辑器使用指南

## 概述
路线编辑器用于创建和编辑龙的飞行路线。路线由一系列点组成，龙会按顺序沿着这些点飞行。

## 坐标系统
- **游戏坐标**：(0, 0) 到 (360, 640)，这是路线存储和游戏逻辑使用的坐标
- **画布坐标**：(0, 0) 到 (640, 800)，这是实际画布像素坐标
- **偏移量**：`CANVAS_OFFSET_X = 140`, `CANVAS_OFFSET_Y = 100`
- **转换公式**：
  - 游戏坐标 → 画布坐标：`canvasX = gameX + 140`, `canvasY = gameY + 100`
  - 画布坐标 → 游戏坐标：`gameX = canvasX - 140`, `gameY = canvasY - 100`

## 使用步骤

### 1. 进入路线编辑模式
- 在游戏开始画面，点击"绘制路线"按钮

### 2. 新建路线
- 点击底部 **"新建"** 按钮（第一个按钮）
- 画布中间的虚线框内会出现一个绿色圆点（起点）

### 3. 绘制路线
- 在虚线框内按住鼠标左键（或触摸屏幕）并拖动
- 移动鼠标时，路线会实时显示
- 松开鼠标完成绘制

### 4. 编辑现有路线
- 点击底部 **"选择"** 按钮（如果需要切换路线）
- 点击路线编号来选择要编辑的路线

### 5. 优化路线
- 点击底部 **"优化"** 按钮
- 系统会自动优化路线，使其更平滑

### 6. 保存路线
- 点击底部 **"保存"** 按钮
- 路线会保存到浏览器的 localStorage 中

### 7. 导出路线
- 点击底部 **"导出"** 按钮
- 路线会导出为 JSON 文件，可以分享或备份

### 8. 返回游戏
- 点击底部 **"返回"** 按钮
- 返回到游戏开始画面

## 按钮说明

底部按钮栏共有 6 个按钮：
1. **新建**：创建一条新路线
2. **清除**：清除当前选中的路线
3. **保存**：保存所有路线到 localStorage
4. **优化**：优化当前路线（平滑处理）
5. **导出**：导出所有路线到 JSON 文件
6. **返回**：返回游戏开始画面

## 路线图例

- 🟢 **绿色圆点**：路线起点
- 🔴 **红色圆点**：路线终点
- **彩色线条**：路线轨迹（不同路线用不同颜色）
- **虚线**：非当前选中的路线

## 高级功能

### 多点路线
- 可以创建多条路线（点击"新建"多次）
- 游戏中龙会按顺序使用这些路线

### 路线优化
- 点击"优化"按钮，系统会：
  1. 使用 Douglas-Peucker 算法简化路线（减少点数）
  2. 使用 Catmull-Rom 算法平滑路线（使曲线更圆润）

### 导入路线
- 导出的 JSON 文件可以再次导入
- 点击"导入"按钮（如果有），选择 JSON 文件

## 故障排除

### 问题：无法绘制路线
**原因**：可能没有点击"新建"按钮
**解决**：先点击"新建"按钮创建路线

### 问题：路线显示位置不对
**原因**：坐标系统错误
**解决**：确保使用游戏坐标绘制（在虚线框内绘制）

### 问题：路线太锯齿状
**原因**：绘制时点数太多或太密集
**解决**：点击"优化"按钮平滑路线

## 技术细节

### 路线存储格式
```typescript
interface CustomRoute {
  id: string
  name: string
  points: Array<{ x: number; y: number }>
}
```

### 路线文件位置
- **localStorage 键名**：`dragonShooter_routes`
- **导出文件**：`dragon_routes_${timestamp}.json`

### 路线图元信息
- 路线 ID：自动生成，格式为 `route_${timestamp}`
- 路线名称：自动生成，格式为 `路线 ${index + 1}`
- 点数：不宜超过 200 点（性能考虑）

## 提示
- 绘制路线时，尽量平滑移动鼠标，避免剧烈抖动
- 路线起点最好在画布外（y < 0），这样龙会从外面飞入
- 路线终点最好在画布外（y > 640），这样龙会飞出画布
- 可以创建多条路线，增加游戏多样性
