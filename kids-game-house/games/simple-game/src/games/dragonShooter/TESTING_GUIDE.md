## ✅ 测试清单

### 测试1：道具卡牌文字方向

**步骤：**
1. 点击"开始闯关"
2. 射击带有特殊颜色的龙节段（紫色、蓝色等，这些是道具节段）
3. 当道具节段被击杀时，会弹出3张卡牌选择界面
4. 点击任意一张卡牌进行翻转

**预期结果：**
- ✅ 卡牌翻转动画流畅
- ✅ 翻转后图标正向显示（不颠倒）
- ✅ 卡牌名称文字正向显示
- ✅ 描述文字正向显示
- ✅ 所有文字清晰可读

**如果失败：**
- ❌ 文字上下颠倒 → 检查 renderer.ts 第1055-1083行的 Y轴翻转代码
- ❌ 文字模糊 → 检查字体大小和阴影设置

### 测试2：龙的动态加速机制

**步骤：**
1. 点击“开始闯关”
2. 观察龙的生成位置（应该在屏幕上方边缘附近，y≈-50）
3. 等待龙开始移动并逐渐离开屏幕
4. 当龙的大部分身体离开屏幕时，观察控制台日志和龙的移动速度

**预期结果：**

#### 起始位置验证
- ✅ 龙从屏幕上方边缘附近开始（y坐标在 -50 左右）
- ✅ 控制台显示：`🐉 龙起始位置: (180, -50), 原始Y: -200`
- ✅ 玩家可以清楚看到龙的生成过程

#### 阶段1：触发加速
- ✅ 当龙的**未显示部分 > 50%** 时（即超过一半身体在屏幕外）
- ✅ 控制台显示：`🚀 龙 #X 触发加速！未显示比例: XX.X%, 可见节段: N/M`
- ✅ 龙的移动速度明显加快（约2倍速）

#### 阶段2：加速持续
- ✅ 加速持续 1.5 秒
- ✅ 期间龙保持高速移动

#### 阶段3：进入冷却
- ✅ 1.5秒后控制台显示：`⏱️ 龙 #X 加速结束，进入冷却`
- ✅ 龙的速度恢复正常
- ✅ 接下来的5秒内不会再次加速

#### 阶段4：状态重置
- ✅ 如果龙的身体重新进入屏幕，未显示比例 < 50%
- ✅ 加速状态立即取消
- ✅ 下次离开屏幕时可以重新触发

**验证不同场景：**

| 场景 | 操作 | 预期行为 |
|------|------|----------|
| 减速状态 | 使用冰霜道具减速龙 | ❌ 不应该触发加速 |
| 多条龙 | 同时有2-3条龙在屏幕上 | ✅ 每条龙独立管理加速状态 |
| 快速进出 | 龙频繁进出屏幕 | ✅ 状态正确重置，无异常 |
| 游戏暂停 | 按P键暂停游戏 | ✅ 计时器应该停止 |

## 🐛 常见问题排查

### 问题1：卡牌文字仍然倒置

**可能原因：**
- 浏览器缓存未清除
- 代码修改未生效

**解决方法：**
```bash
# 清除 Vite 缓存
rm -rf node_modules/.vite
npm run dev
```

或在浏览器中强制刷新：`Ctrl + Shift + R` (Windows) / `Cmd + Shift + R` (Mac)

### 问题2：龙没有触发加速

**检查点：**
1. 控制台是否有错误信息？
2. 龙的 `_boostTimer` 等字段是否正确初始化？
3. 可视比例计算是否正确？

**调试代码：**
在 `dragon.ts` 的 `updateDragon` 函数开头添加：
```typescript
console.log('龙 #', dragon.id, '可视比例:', visibleRatio, '加速状态:', dragon._isBoosting)
```

### 问题3：加速后立即再次触发

**可能原因：**
- 冷却时间未正确设置
- 状态重置逻辑有问题

**检查：**
确保 `dragon._boostCooldown = 5.0` 在加速结束时被正确执行。

## 📊 性能监控

### FPS 检查

游戏右上角的性能监控会显示：
- FPS（帧率）
- 实体数量（龙、子弹、粒子等）

**正常范围：**
- FPS: 55-60
- 龙数量: 1-4（取决于关卡）
- 粒子数量: < 100

### 内存检查

在浏览器开发者工具的 Performance 或 Memory 标签页：
- 观察内存使用是否稳定
- 检查是否有内存泄漏
- 对象池应该有效减少GC频率
