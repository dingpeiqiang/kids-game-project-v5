# Bubble Shooter 渲染层级修复说明

## 🎯 问题描述

**用户反馈**: "射出来的球不应噶穿过 显示在底部"

**问题分析**: 
- 射出的泡泡在发射器之后绘制
- 导致泡泡会覆盖在发射器上面
- 视觉上看起来泡泡从发射器"穿透"出来，不自然

## ✅ 解决方案

调整渲染顺序，让发射器在飞行泡泡之后绘制，这样发射器会覆盖住刚从底部射出的泡泡。

### 修改前的渲染顺序

```typescript
1. 棋盘泡泡
2. 瞄准线
3. 发射器          ← 先绘制
4. 下一个泡泡预览
5. 飞行中的泡泡     ← 后绘制（会覆盖发射器）❌
6. 粒子效果
7. 漂浮分数
8. UI 信息
```

**问题**: 飞行泡泡在第5步绘制，会覆盖在第3步绘制的发射器上，导致泡泡看起来从发射器中"穿透"出来。

### 修改后的渲染顺序

```typescript
1. 棋盘泡泡
2. 飞行中的泡泡     ← 先绘制 ✅
3. 瞄准线
4. 发射器          ← 后绘制（覆盖飞行泡泡）✅
5. 下一个泡泡预览
6. 粒子效果
7. 漂浮分数
8. UI 信息
```

**优势**: 
- 发射器在第4步绘制，会覆盖在第2步绘制的飞行泡泡上
- 当泡泡刚从发射器射出时，发射器会遮住泡泡的底部
- 视觉效果更自然，像是从发射器"发射"出来

## 🔧 代码修改

### 文件: Renderer.ts

```typescript
// 修改前
render() {
  // ... 背景和棋盘
  
  this.drawBubbles(ctx, board)        // 棋盘泡泡
  this.drawAimLine(ctx, shooter)      // 瞄准线
  this.drawShooter(ctx, shooter)      // 发射器
  this.drawNextBubblePreview(ctx)     // 下一个预览
  if (projectile) {
    this.drawProjectile(ctx, projectile)  // ❌ 飞行泡泡（最后）
  }
  // ... 粒子和UI
}

// 修改后
render() {
  // ... 背景和棋盘
  
  this.drawBubbles(ctx, board)        // 棋盘泡泡
  if (projectile) {
    this.drawProjectile(ctx, projectile)  // ✅ 飞行泡泡（提前）
  }
  this.drawAimLine(ctx, shooter)      // 瞄准线
  this.drawShooter(ctx, shooter)      // 发射器（覆盖飞行泡泡）
  this.drawNextBubblePreview(ctx)     // 下一个预览
  // ... 粒子和UI
}
```

## 🎨 Canvas 渲染层级原理

Canvas 使用**画家算法**（Painter's Algorithm）：
- **后绘制的元素**会覆盖**先绘制的元素**
- 类似于一层一层往上画
- 最后画的内容在最上面

### 正确的层级关系

```
最上层 (最后绘制)
├── UI 信息 (分数、时间)
├── 漂浮分数
├── 粒子效果
├── 下一个泡泡预览
├── 发射器              ← 覆盖飞行泡泡
├── 瞄准线
├── 飞行中的泡泡         ← 在发射器下面
├── 棋盘泡泡
├── 棋盘背景
└── 星空背景
最下层 (最先绘制)
```

## 💡 设计考虑

### 为什么这样安排？

1. **棋盘泡泡** - 最早绘制，作为游戏主体
2. **飞行泡泡** - 在发射器之前，避免穿透
3. **瞄准线** - 在泡泡之上，清晰可见
4. **发射器** - 覆盖飞行泡泡起点，视觉自然
5. **下一个预览** - 在发射器之上，提示玩家
6. **粒子/分数/UI** - 最上层，确保可见

### 特殊情况处理

