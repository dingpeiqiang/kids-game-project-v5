# 路线编辑器按钮调试指南

## 🐛 问题描述

清除、保存、导出按钮点击没有反应。

## 🔍 调试步骤

### 1. 打开浏览器控制台

按 F12 打开开发者工具，切换到 Console 标签。

### 2. 进入路线编辑器

1. 点击"✏️ 绘制路线"按钮
2. 在画布上绘制一些点（至少3个）

### 3. 点击按钮并查看日志

#### 期望看到的日志：

```
🖱️ handleDown: { x: 400, y: 670, phase: 'routeEdit' }
🎯 按钮区域: { btnStartX: 295, btnY: 670, btnW: 85, btnH: 50, totalWidth: 355 }
✅ 点击在按钮Y范围内
🔘 点击了按钮区域 { x: 400, y: 670, btnStartX: 295, btnY: 670 }
🗑️ 点击清除按钮 { x: 400, btnStartX: 295, endX: 380 }
```

或者

```
💾 点击保存按钮 { x: 485, startX: 385, endX: 475 }
📍 当前点数: 5
```

或者

```
📥 点击导出按钮 { x: 575, startX: 480, endX: 570 }
📍 当前点数: 5
```

### 4. 可能的问题和解决方案

#### 问题A：没有看到 "🎯 按钮区域" 日志

**原因：** 点击坐标不在按钮的Y范围内

**检查：**
- 查看 `handleDown` 日志中的 y 值
- 按钮Y范围是 `btnY` 到 `btnY + btnH`（通常是 670-720）
- 如果 y < 670 或 y > 720，说明点击位置不对

**解决：** 确保点击在屏幕底部的按钮区域

---

#### 问题B：看到 "✅ 点击在按钮Y范围内" 但没有按钮日志

**原因：** X坐标不在任何按钮范围内

**检查：**
- 查看日志中的 x 值
- 按钮X范围：
  - 清除按钮：295-380
  - 保存按钮：385-475
  - 导出按钮：480-570
  - 返回按钮：575-665

**解决：** 确保点击在正确的按钮上

---

#### 问题C：看到按钮日志但显示 "❌ routeEditor 未初始化!"

**原因：** routeEditor 对象没有正确创建

**解决：**
1. 刷新页面
2. 重新进入路线编辑器
3. 检查是否有其他错误

---

#### 问题D：看到按钮日志但没有后续动作

**原因：** 
- 清除：应该清空画布
- 保存：需要至少3个点
- 导出：需要至少3个点，会触发文件下载

**检查：**
- 是否绘制了足够的点？
- 浏览器是否阻止了弹窗/下载？

## 🎯 快速测试

### ⚠️ 重要提示：绘制路线的正确方式

**路线编辑器需要拖动鼠标来绘制，不是点击！**

1. **开始绘制：**
   - 在画布空白处**按下鼠标左键**
   - **不要松开**，拖动鼠标
   - 你会看到点被添加

2. **结束绘制：**
   - 松开鼠标左键
   - 或者点击按钮

3. **测试清除按钮：**
   - 先绘制一些点（拖动鼠标）
   - 点击红色"清除"按钮
   - 应该清空所有点

4. **测试保存按钮：**
   - 先绘制至少3个点（拖动鼠标）
   - 点击绿色"保存"按钮
   - 应该显示 "✅ 路线已保存!"

5. **测试导出按钮：**
   - 先绘制至少3个点（拖动鼠标）
   - 点击蓝色"导出"按钮
   - 应该下载 JSON 文件

6. **测试返回按钮：**
   - 点击橙色"返回"按钮
   - 应该回到开始界面
