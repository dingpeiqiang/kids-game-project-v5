# 🐛 调试面板系统使用指南

## 📋 功能概述

调试面板系统包含两个组件：
1. **玩家调试面板** - 实时显示玩家的游戏状态和属性信息（左上角）
2. **实体调试面板** - 通用实体监控，支持玩家、敌人等多种实体（右上角）

## 🎯 如何启用

### 方法一：顶部工具栏按钮（推荐）
- **点击屏幕顶部中间**的 `🔍 调试监控` 按钮
- 按钮会变为红色背景并显示 `❌ 关闭调试`
- 再次点击即可关闭

### 方法二：代码控制
```typescript
// 获取调试面板引用
const playerPanel = (this.scene as any).playerDebugPanel
const entityPanel = (this.scene as any).entityDebugPanel

// 显示面板
playerPanel.show()
entityPanel.show()

// 隐藏面板
playerPanel.hide()
entityPanel.hide()

// 切换显示状态
playerPanel.toggle()
entityPanel.toggle()
```

## 📊 显示信息详解

### 🎮 玩家调试面板（左上角）

#### 📍 基础属性
- **位置** - 玩家在游戏世界中的坐标 (x, y)
- **速度** - 玩家的移动速度向量 (velocityX, velocityY)
- **生命** - 玩家当前生命值
- **护甲** - 玩家当前护甲层数

#### 🎯 战斗状态
- **护盾** - 护盾道具是否激活（✅ 已激活 / ❌ 未激活）
- **无敌** - 是否处于无敌状态（✅ 是 / ❌ 否）
- **状态** - 玩家状态机当前状态（ALIVE/DYING/RESPAWNING/DEAD）
- **生命数** - 剩余复活次数

#### 🎮 移动状态
- **方向** - 当前移动方向（UP/DOWN/LEFT/RIGHT/NONE）
- **移动中** - 是否正在移动（✅ 是 / ❌ 否）

#### 🔫 战斗能力
- **子弹伤害** - 当前子弹伤害值
- **射速** - 射击冷却时间（毫秒）
- **可射击** - 是否可以射击（✅ 可以 / ❌ 冷却中）

#### 🎨 渲染状态
- **可见** - 玩家是否可见（✅ 可见 / ❌ 不可见）
- **透明度** - Alpha 透明度值（0.00-1.00）
- **激活** - 实体是否激活（✅ 是 / ❌ 否）
- **纹理** - 当前使用的纹理名称

#### ⚙️ 道具效果
- **冻结** - 是否被冻结（✅ 是 / ❌ 否）
- **加速** - 是否处于加速状态（✅ 是 / ❌ 否）

---

### 🤖 实体调试面板（右上角）

实时监控战场上所有活跃敌人，每个敌人显示：

#### 📦 通用信息
- **🎮/🤖 名称和类型** - 实体名称 + 类型图标
- **📍 位置** - 游戏世界坐标
- **➡️ 速度向量** - 移动方向和速度
- **❤️ 生命值** - 当前剩余生命

#### 🤖 敌人特有属性
- **🤖 类型** - 敌人类型（LIGHT/MEDIUM/HEAVY）
- **⚔️ 伤害** - 敌人攻击力
- **💨 速度** - 移动速度
- **👁️ 可见性** - 是否可见
- **🌟 透明度** - Alpha 值

## 🔍 典型使用场景

### 场景 1：验证护盾消耗后玩家可见性（玩家面板）
1. 获得护盾道具
2. **点击工具栏按钮**打开调试面板
3. 被子弹击中
4. 观察玩家面板中 **"护盾"** 从 ✅ 变为 ❌
5. 确认 **"可见"** 保持 ✅

### 场景 2：检查复活后的无敌状态（玩家面板）
1. 玩家死亡并复活
2. 观察 **"状态"** 从 DYING → RESPAWNING → INVINCIBLE → ALIVE
3. 确认 **"无敌"** 在 INVINCIBLE 状态下为 ✅
4. 2 秒后自动变为 ❌

### 场景 3：调试玩家消失问题（玩家面板）
1. 按快捷键打开对应面板
2. 如果玩家不可见但 **"激活"** 为 ✅
3. 检查 **"可见"** 和 **"透明度"**
4. 如果 **"可见"** 为 ❌ 或 **"透明度"** 为 0，说明是渲染问题

### 场景 4：验证道具效果（玩家面板）
1. 吃到加速道具
2. 观察 **"加速"** 变为 ✅
3. 检查速度值应该 > 200
4. 效果结束后自动变回 ❌

### 场景 5：监控敌人数量和状态（实体面板）
1. **点击工具栏按钮**打开实体调试面板
2. 观察战场上所有活跃敌人
3. 每个敌人的生命值、类型、位置一目了然
4. 敌人死亡后自动从列表中移除

### 场景 6：调试敌人 AI 行为（实体面板）
1. **点击工具栏按钮**打开实体调试面板
2. 观察特定敌人的移动轨迹和状态变化
3. 结合控制台日志分析 AI 决策

## 💡 调试技巧

### 工具栏特性
- **悬浮设计** - 固定在屏幕顶部中间，不遮挡游戏画面
- **居中布局** - 水平居中，距离顶部 10px
- **鼠标悬停** - 高亮显示，鼠标指针变为手型
- **点击反馈** - 按钮有缩放动画反馈
- **状态指示** - 开启时背景变红显示 `❌ 关闭调试`
- **快捷操作** - 一键切换所有调试面板

### 性能优化
- **玩家面板** - 每 100ms 更新一次
- **实体面板** - 每 100ms 更新一次，最多显示 20 个敌人
- **敌人监控** - 每 500ms 扫描一次活跃敌人
- 只在需要时开启，测试完成后关闭

### 日志配合
调试面板与 console.log 配合使用效果更佳：
```typescript
// 控制台输出详细日志
console.log('🔍 [调试] onHit 检查:', { ... })

// 玩家面板实时显示玩家状态
// 实体面板实时显示敌人状态
// 按 F3 查看可视化信息
```

### 问题排查流程
1. **点击工具栏按钮**开启对应面板
2. **复现问题**（如玩家/敌人消失、不受控制等）
3. **观察相关字段**（可见性、状态、方向等）
4. **结合控制台日志**定位问题
5. **修复后验证**所有相关状态

## 🛠️ 技术实现

### 架构设计
```
TankGameScene
├── playerDebugPanel: PlayerDebugPanel
│   ├── 只显示玩家属性
│   └── update(time): void
├── entityDebugPanel: EntityDebugPanel
│   ├── 显示所有监控实体
│   ├── addEntity(id, data): void
│   ├── removeEntity(id): void
│   └── update(time): void
└── enemyDebugManager: EnemyDebugManager
    ├── 自动扫描活跃敌人
    ├── 添加/移除监控
    └── update(time): void
```

### 关键特性
- **双层面板** - 玩家专用 + 通用实体监控
- **自动管理** - 敌人自动生成/死亡自动移除
- **高性能** - 限制更新频率和显示数量
- **可插拔** - 随时开启/关闭
- **深度优先** - 始终显示在最上层（depth: 9999）

## 📝 注意事项

1. **仅用于调试** - 生产环境应关闭调试面板
2. **不依赖此功能** - 游戏逻辑不应依赖调试面板的状态
3. **及时清理** - 场景关闭时会自动销毁
4. **性能考虑** - 
   - 玩家面板每帧更新（100ms 间隔）
   - 实体面板每帧更新（100ms 间隔，最多 20 个敌人）
   - 敌人监控每 500ms 扫描一次
5. **显示限制** - 实体面板最多同时显示 20 个敌人
