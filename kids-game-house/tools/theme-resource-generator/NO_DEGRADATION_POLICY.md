# 🚫 无降级交付 - 严格执行政策

**版本**: v1.0  
**最后更新**: 2026-03-27  
**适用范围**: 所有游戏主题资源生成项目

---

## ⚠️ 核心原则

> **整体交付不允许降级！**

这意味着：
- ✅ **100% 满足 GDD 需求** - 缺一不可
- ✅ **真实可用资源** - 不允许占位符
- ✅ **严格质量校验** - 不合格就报错
- ✅ **零容忍政策** - 发现降级直接失败

---

## 📋 什么是降级方案？

### ❌ 禁止的降级行为

#### 1. 图片资源降级

- ❌ 使用 Sharp 生成的简单几何图形
- ❌ 纯色块、圆形、方形等
- ❌ 分辨率低于标准（<256x256）
- ❌ 文件过小（<100 字节）

**示例**:
```xml
<!-- ❌ 错误：简单的三角形代表飞机 -->
<svg>
  <polygon points="128,32 192,224 64,224" fill="#green"/>
</svg>
```

#### 2. 音频资源降级

- ❌ 空文件（0 字节）
- ❌ 静音文件
- ❌ 时长不足（音效<0.1 秒，BGM<1 分钟）
- ❌ 文件过小（<1KB）

**示例**:
```bash
# ❌ 错误：创建空 MP3 文件
touch sfx_shoot.mp3  # 0 字节
```

#### 3. 配置文件降级

- ❌ 缺少必需字段
- ❌ 路径指向不存在的文件
- ❌ 格式错误无法解析

## ✅ 什么是合格交付？

### 图片资源标准

| 检查项 | 标准要求 | 检测方法 |
|--------|---------|---------|
| **文件格式** | PNG / JPG | 扩展名检查 |
| **文件大小** | ≥100 字节 | `fs.stat().size` |
| **图片尺寸** | ≥256x256 | sharp.metadata() |
| **内容质量** | 有实际图案 | TODO: AI 检测 |
| **透明度** | PNG 支持透明 | sharp.hasAlpha() |

**合格示例**:
- ✅ 精心绘制的玩家飞机（有细节、有层次）
- ✅ 设计精美的敌机（有眼睛、有纹理）
- ✅ 专业的子弹效果（带渐变、发光）
- ✅ 美观的道具图标（有文字、有边框）

### 音频资源标准

| 检查项 | 标准要求 | 检测方法 |
|--------|---------|---------|
| **文件格式** | MP3 / WAV | 扩展名检查 |
| **文件大小** | ≥1KB (音效), ≥100KB (BGM) | `fs.stat().size` |
| **时长** | ≥0.1 秒 (音效), ≥60 秒 (BGM) | ffprobe / music-metadata |
| **比特率** | ≥128 kbps | music-metadata |
| **采样率** | 44.1 kHz | music-metadata |
| **内容质量** | 非静音 | TODO: 波形分析 |

**合格示例**:
- ✅ 真实的射击音效（录制或专业库）
- ✅ 震撼的爆炸音效（多层叠加）
- ✅ 欢快的背景音乐（原创或授权）
- ✅ 清脆的道具拾取音（清晰可辨）

## 🔍 如何确保无降级

### 工具层面

#### 1. 严格校验器已升级

```javascript
// strict-validator.js

// 检查空文件（零容忍）
if (stats.size === 0) {
  errors.push('❌ 文件大小为 0（占位符/空文件，违反无降级交付原则）');
}

// 检查最小文件大小
const minSizeMap = {
  '.png': 100,      // PNG 至少 100 字节
  '.mp3': 1000,     // MP3 至少 1KB
};

if (stats.size < minSize) {
  errors.push(`❌ 文件过小 (${stats.size} bytes)，疑似降级方案`);
}
```

#### 2. 资源生成器已禁用占位符

```javascript
// resource-generator.js

// ❌ 旧代码：生成空文件
await fs.writeFile(filepath, Buffer.from([]));

// ✅ 新代码：必须从模板或 API 生成
const generated = await this.generateRealAudio(audioReq);
if (!generated) {
  result.hasEmptyFiles = true; // 标记失败
  console.error(`❌ 无法生成音频：${audioReq.name}`);
}
```

#### 3. 自动检测降级方案

```javascript
// validateGeneratedResources()

// 检查是否有空文件
if (result.hasEmptyFiles) {
  failures.push({
    name: '空文件检查',
    reason: '发现空文件（占位符），不符合无降级交付要求'
  });
}

// 任何一项失败都会导致整体失败
return {
  passed: failures.length === 0,
  failures
};
```

### 流程层面

#### 1. GDD 评审阶段

**检查清单**:
- [ ] 所有资源需求明确且合理
- [ ] 有对应的音频/图片来源规划
- [ ] 团队确认能提供所有资源
- [ ] 移除无法实现的资源需求

#### 2. 资源准备阶段

**选项排序**:
1. ✅ **使用模板**: 从 templates/audio/ 复制
2. ✅ **外部采购**: 购买版权音乐/音效
3. ✅ **自己制作**: 录制或编辑
4. ✅ **AI 生成**: 使用 AI 工具创作
5. ❌ **降级方案**: 直接报错，不接受

#### 3. 生成验证阶段

**运行命令**:
```bash
npm run generate -- -g GDD.md -o output -t theme
npm run validate -- -g GDD.md -r output
```

**输出检查**:
```
✅ 通过: "所有资源验证通过！"
❌ 失败: "❌ 缺失资源：sfx_shoot.mp3"
         "❌ 文件过小：player.png (50 bytes)"
```

#### 4. 交付审核阶段

**审核要点**:
- [ ] 所有资源文件存在且大小合理
- [ ] 图片内容清晰可辨认
- [ ] 音频可以正常播放
- [ ] GTRS 配置正确无误

## 🛠️ 实施建议

### 对于新项目

**从零开始**:

```bash
# 1. 编写 GDD（确保需求可实现）
vim GAME_DESIGN_DOCUMENT.md

# 2. 准备音频模板
cp /path/to/audio/*.mp3 templates/audio/

# 3. 生成资源
npm run generate -- -g GAME_DESIGN_DOCUMENT.md -o output -t theme

# 4. 严格验证
npm run validate -- -g GAME_DESIGN_DOCUMENT.md -r output

# 5. 如果有失败，解决后重新生成和验证
# 绝不允许带着降级方案上线！
```

### 对于已有项目

**清理历史债务**:

```bash
# 1. 检查现有资源
npm run validate -- -g GDD.md -r existing-resources

# 2. 列出不合格的资源
# - 空文件
# - 过小的文件
# - 缺失的文件

# 3. 制定替换计划
# - 优先级 1: 关键资源（玩家、敌人）
# - 优先级 2: 重要资源（子弹、道具）
# - 优先级 3: 次要资源（装饰、背景）

# 4. 逐个替换为真实资源
# 每替换一个，验证一次

# 5. 全部替换完成后，重新生成
npm run generate -- -g GDD.md -o new-output -t theme-v2
```

## 📊 对比数据

### 降级交付 vs 无降级交付

| 维度 | 降级交付 | 无降级交付 |
|------|---------|-----------|
| **用户体验** | 差（简陋的几何图形） | 好（精美的美术资源） |
| **开发效率** | 低（后期要返工） | 高（一次做到位） |
| **维护成本** | 高（技术债务积累） | 低（质量稳定） |
| **团队协作** | 难（标准不统一） | 易（有明确规范） |
| **产品质量** | 参差不齐 | 始终如一 |

### 短期痛苦 vs 长期收益

```
降级交付:
短期：快速完成 ✅
长期：反复返工、质量低下 ❌

无降级交付:
短期：投入更多时间准备资源 ⏰
长期：一次到位、质量可靠 ✅
```

## 🎯 成功案例

### 案例 1: 飞机大战（严格执行）

**做法**:
1. 先从 Freesound 下载音效模板
2. 请美术设计师绘制玩家飞机和敌机

4. 发现 sfx_explosion.mp3 过小（800 字节）
5. 立即替换为真实的爆炸音效（50KB）
6. 重新验证通过后才交付

**结果**:
- ✅ 所有图片≥256x256，内容丰富
- ✅ 所有音效≥1KB，清晰可辨
- ✅ BGM 2 分 30 秒，循环流畅
- ✅ 用户评价："专业！"

### 案例 2: 坦克大战（初期降级，后期返工）

**初期（降级方案）**:
- ❌ 玩家坦克 = 绿色方块
- ❌ 敌人坦克 = 红色圆圈
- ❌ 射击音效 = 空文件
- ❌ 背景音乐 = 缺失

**问题暴露**:
- 测试反馈："太简陋了，像半成品"
- 开发人员："不好意思给别人看"
- 产品经理："这怎么上线？"

**后期（整改）**:
1. 停止开发新功能
2. 专心准备真实资源
3. 重新生成并验证
4. 达到无降级标准

**教训**:
> **早期降级 = 后期返工**
> 
> **一次做到位 = 省时省力**

## ⚡ 常见问题

### Q1: 暂时没有美术资源怎么办？

**A**: 
1. **选项 1**: 找兼职美术设计师（推荐）
2. **选项 2**: 使用开源游戏资源包
3. **选项 3**: 简化 GDD，减少资源需求
4. ❌ **不可以**: 用几何图形代替

### Q2: 音频太难找了怎么办？

**A**: 
1. **Freesound.org** - 大量免费音效
2. **Kenney.nl** - CC0 游戏音效包
3. **OpenGameArt** - 专门的游戏资源站
4. ❌ **不可以**: 用空文件占位

### Q3: 能不能先降级，后期再替换？

**A**: **绝对不能！**

原因:
- 降级的代码会污染项目
- 团队成员会降低标准
- 后期替换成本更高
- 用户第一印象很重要

**正确做法**:
- 要么不做了
- 要么一次做到位
- 没有中间地带

### Q4: 如何判断是否降级？

**A**: 问自己三个问题:

1. **这个资源敢给用户看吗？**
   - 敢 → 可能合格
   - 不敢 → 肯定是降级

2. **这个资源符合商业质量标准吗？**
   - 符合 → 合格
   - 不符合 → 降级

3. **我愿意把这个资源展示给投资人/老板吗？**
   - 愿意 → 可能合格
   - 不愿意 → 降级
