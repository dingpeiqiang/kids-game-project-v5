## 🎯 遵循开发规范

### 核心理念践行
✅ **最大化聚焦游戏内容**  
- AI 专注于游戏玩法设计（移动、射击、碰撞）
- 没有重新实现 UI 组件、状态管理、路由配置

✅ **贪吃蛇代码已验证可用**  
- 直接复制了 snake 目录结构
- 复用了 PhaserGame.ts 框架层代码
- 保留了 GTRS 加载、屏幕适配等通用功能

✅ **最小化改动适配**  
- 只修改了 StartView 的标题和描述
- 重写了 PhaserGame.ts 的游戏逻辑部分（第 600 行以后）
- 其他文件保持不变

✅ **工具化自动化**  
- 使用 gtrs-generator 自动生成资源
- Node.js 脚本生成 PNG 和 WAV 文件
- 避免手动绘制图片和录制音频

✅ **规范化配置**  
- 严格按照 GTRS 规范组织资源
- 统一的目录结构
- 标准化的 SQL 注册脚本

## 🎨 游戏特色

### 视觉设计
- 🌌 **太空主题**: 深蓝渐变背景 + 星空点缀
- ✈️ **流线型飞机**: 绿色战斗机，造型简洁
- 👾 **多样化敌机**: 3 种类型，颜色区分明显
- 💥 **炫酷爆炸**: 4 帧动画，从白到红的渐变

### 操作手感
- 🎮 **双模式控制**: 键盘 + 鼠标/触摸
- 🚀 **平滑移动**: 基于时间的运动，不受帧率影响
- ⚡ **快速响应**: 输入延迟 < 16ms
- 🔫 **连发射击**: 自动射击，无需频繁按键

### 难度平衡
- 📈 **渐进式难度**: 从慢到快，循序渐进
- 🎯 **公平挑战**: 敌机生成位置随机，避免无解局面
- ⏱️ **无敌时间**: 受伤后 2 秒保护，降低挫败感

## 📝 部署步骤

### 1. 安装依赖
```bash
cd kids-game-house/games/plane-shooter
npm install
```

### 2. 启动开发服务器
```bash
npm run dev
```

### 3. 注册游戏到数据库
```bash
mysql -u root -p kids_game_platform < games/plane-shooter/public/register-game.sql
```

### 4. 验证访问
浏览器打开：`http://localhost:8085`
