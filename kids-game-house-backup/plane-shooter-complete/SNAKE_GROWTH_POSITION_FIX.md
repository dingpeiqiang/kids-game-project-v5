# 蛇身增长位置修复 - Phase 3 完善

## 🐛 问题描述

**症状**：蛇吃到食物变长后，新增加的节都堆叠在同一个位置 `(528.7, 349.5)`。

**日志显示**：
```
🐍 蛇身位置：
节 0-22: 形成漂亮的 S 形曲线 ✅
节 23: (528.7, 349.5)
节 24: (528.7, 349.5)  ❌
节 25: (528.7, 349.5)  ❌
...
节 73: (528.7, 349.5)  ❌
```

**分析**：
- ✅ 前 23 节正常工作（距离约束生效）
- ❌ 从第 23 节开始，所有新增节都在同一位置
- ❌ 这些节没有参与距离约束跟随

## 🔍 根本原因

### 错误的增长逻辑

```typescript
// ❌ 之前的实现
const lastSegment = snake[snake.length - 1]
snake.push({ ...lastSegment })  // 复制最后一节的位置
```

**问题**：
1. 新节与最后一节在**同一位置**
2. 距离 = 0，距离约束不会触发
3. 下一帧移动时：
   - 蛇头移动
   - 第 1 节被拉向蛇头
   - 第 2 节被拉向第 1 节
   - ...
   - 第 22 节被拉向第 21 节
   - **第 23 节及以后**：因为距离=0，不会被拉动！

### 为什么会堆叠？

```
初始状态（吃食物前）：
[节 21] ---45px--- [节 22]

吃食物瞬间：
[节 21] ---45px--- [节 22] [节 23]  ← 在同一位置
                      ↑
                  距离=0，不触发约束

第 1 帧移动：
[节 21 移动到新位置]
    |
    | 距离 > 45px，触发约束
    ↓
[节 22 被拉向节 21]
[节 23 仍在原地]  ← 距离=0，不触发约束

第 2 帧移动：
...
[节 23 仍然在原地]  ← 永远不动！
```

## ✅ 解决方案

### 正确的增长逻辑

```typescript
// ✅ 新的实现
const lastSegment = snake[snake.length - 1]
const secondLastSegment = snake[snake.length - 2]

if (secondLastSegment) {
  // 计算尾巴延伸方向
  const dx = lastSegment.x - secondLastSegment.x
  const dy = lastSegment.y - secondLastSegment.y
  
  // 新节在尾巴后面，保持相同方向和距离
  const newSegment = {
    x: lastSegment.x + dx,
    y: lastSegment.y + dy
  }
  snake.push(newSegment)
} else {
  // 只有一节时（罕见情况），直接复制
  snake.push({ ...lastSegment })
}
```

**原理**：
```
原尾巴：
[节 21] ---45px--- [节 22]
                    ↑
                延长相同方向
                    ↓
新尾巴：
[节 21] ---45px--- [节 22] ---45px--- [节 23]
```

### 数学原理

```typescript
// 向量法：从节 21 指向节 22 的向量
vector = (节 22.x - 节 21.x, 节 22.y - 节 21.y)

// 新节位置 = 节 22 + vector
newSegment = (节 22.x + vector.x, 节 22.y + vector.y)

// 结果：三节在一条直线上，间距相等
```

**可视化**：
```
● 节 21 (x1, y1)
 |
 | vector = (dx, dy)
 |
● 节 22 (x2, y2) = (x1+dx, y1+dy)
 |
 | 同样的 vector
 |
● 节 23 (x3, y3) = (x2+dx, y2+dy)
```

## ⚠️ 注意事项

### 为什么不用单位向量？

```typescript
// ❌ 复杂方案：单位向量
const length = Math.sqrt(dx*dx + dy*dy)
const unitX = dx / length
const unitY = dy / length
const newSegment = {
  x: lastSegment.x + unitX * segmentDistance,
  y: lastSegment.y + unitY * segmentDistance
}

// ✅ 简单方案：直接用向量
const newSegment = {
  x: lastSegment.x + dx,
  y: lastSegment.y + dy
}
```

**原因**：
- 最后两节已经相距约 `segmentDistance`
- 直接用向量 ≈ 单位向量 × distance
- 更简单，更快

### 如果最后两节距离不等于 segmentDistance？

```typescript
// 情况 1：距离 < segmentDistance（刚转弯）
// 新节会延续这个较小距离
// 下帧距离约束会调整
✅ 自动修正

// 情况 2：距离 > segmentDistance（快速转向）
// 新节会延续这个较大距离
// 下帧距离约束会拉回
✅ 自动修正
```
