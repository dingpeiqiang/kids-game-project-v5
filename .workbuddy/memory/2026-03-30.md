## 修复 Phaser CDN 报错（2026-03-30）

**问题**：`GameScene.ts` 报错 `does not provide an export named 'default'`

**根因**：`package.json` 中虽已移除 phaser，但 `node_modules/phaser` 目录还在（之前 npm install 留下的），Vite 优先找到了本地 `node_modules/phaser` 而非走 CDN 全局变量。`optimizeDeps.exclude` 只阻止预构建，不阻止运行时路径解析。

**解决**：直接删除各项目 `node_modules/phaser` 目录：
- puzzle ✅、snake ✅、plane-shooter ✅、kids-game-frontend ✅（kids-game-frame-factory 本来就没有）

**结论**：仅靠 `optimizeDeps.exclude` 不够，必须同时物理删除 node_modules 中的包。

## 修复 plane-shooter 背景滚动不连续（2026-03-30）

**问题**：飞机大战游戏背景滚动出现跳帧/不连续。

**根因**：`createBackground()` 用 `this.tweens.add` 同时驱动 `bg1`/`bg2`，但两图起点不同（`bg1.y=0`，`bg2.y=-bgHeight`），tween 会独立按距离插值，速度不一致；且 `onUpdate` 里直接修改 `.y` 会被 tween 下一帧覆盖，导致瞬间跳位。

**修复方案**（`PlaneShooterScene.ts`）：
1. 新增成员变量 `bg1`/`bg2`/`bgHeight`/`bgScrollSpeed`
2. `createBackground()` 只负责创建图片并保存引用（移除 tween）
3. 新增 `updateBackground(delta)` 方法：每帧用 `delta/1000 * speed` 推进两图 y 坐标，超出屏幕底部时跳到另一张图的正上方（`this.bg1.y = this.bg2.y - this.bgHeight`）
4. `gameLoop()` 最前面调用 `this.updateBackground(delta)`

## 修复背景图被纯色矩形遮挡（2026-03-30）

**问题**：修复滚动后背景星空图看不见了。

**根因**：上一次修复时给 `bg1`/`bg2` 设了 `setDepth(-10)`，但兜底纯色矩形 `bgColor` 未设 depth（默认 0），导致矩形盖在背景图上面。

**修复**：`bgColor` 改为 `setDepth(-1)`，`bg1`/`bg2` 改为 `setDepth(0)`，层次关系明确。
