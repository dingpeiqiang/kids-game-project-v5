## tank-battle isDying 标志未重置 Bug 修复（2026-03-31 23:43）

**问题现象**：`loadLevel` 显示 `isDying: false`，但 `playerHit()` 的防护检查却触发"玩家已死亡或正在死亡"。

**根本原因**：`loadLevel` 中当玩家存活时，只重置了位置/速度/纹理，但没有重置 `isDying` 和 `isInvincible` 标志。如果玩家之前被击中过（比如护盾被击碎但还有血），`isDying` 仍然为 `true`，导致新关卡第一帧就跳过所有受击逻辑。

**修复**：在 `TankGameScene.ts` 的 `loadLevel` 方法中，玩家存活分支添加：
```typescript
this.isDying = false
this.isInvincible = false
```

**涉及文件**：`kids-game-house/games/tank-battle/src/scenes/TankGameScene.ts`

## tank-battle Framebuffer 修复（2026-03-31 19:20）

**v-show 导致 WebGL Framebuffer Incomplete Attachment**：
- 原因：`v-show` 用 `display: none` 隐藏 canvas，Phaser resize 时尝试创建 0×0 framebuffer
- 修复：改用 `visibility: hidden` + `pointer-events: none`（computed style），canvas 保持正常尺寸
