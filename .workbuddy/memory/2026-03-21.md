## 主题API认证安全加固（2026-03-21 01:10）

### 问题
用户要求对主题管理功能实施严格认证策略：未登录用户必须完全无法操作任何主题相关API，必须登录并携带token才能访问。

### 分析
1. 当前JWT拦截器配置存在矛盾：虽然`ThemeController`类有`@RequireLogin`注解，但WebConfig将主题相关API排除在JWT拦截之外
2. 这导致未登录用户仍然可以访问主题列表、详情等API，违背安全原则
3. JWT拦截器逻辑：只有在类或方法上有`@RequireLogin`注解且路径未被排除时才会进行token验证

### 解决方案
1. **移除主题API的排除项**：
   - 从WebConfig.java中删除所有主题相关API的排除路径（`/api/theme/list`、`/api/theme/detail`等）
   - 现在所有主题API都会经过JWT拦截器，触发`@RequireLogin`注解验证
   
2. **确保认证接口正常工作**：
   - 添加统一认证接口`/api/auth/login`和`/api/auth/public-key`到排除列表
   - 这些接口是登录流程的一部分，必须允许未登录访问

### 修改文件
1. `WebConfig.java`：
   - 移除主题相关API的排除项：`/api/theme/list`, `/api/theme/detail`, `/api/theme/validate-gtrs`, `/api/theme/detect-format`, `/api/theme/migrate-to-gtrs`, `/api/theme/quick-validate`
   - 添加认证接口到排除项：`/api/auth/login`, `/api/auth/public-key`

### 安全效果
现在实现了严格认证模式：
- ✅ 未登录用户无法访问任何主题API（返回401 Unauthorized）

- ✅ 游戏等公共功能仍然可访问（保持原有排除项）

## 主题列表显示问题修复（2026-03-21 01:30）

### 问题描述
主题管理页面无法显示主题列表，虽然接口返回了数据（list 有 6 个主题），但页面显示"暂无主题"。

接口返回数据：
```json
{
  "code": 200,
  "msg": "success",
  "data": {
    "total": 0,  // ❌ 问题：total 为 0
    "pageSize": 20,
    "page": 1,
    "list": [6个主题]  // ✅ 但 list 有数据
  }
}
```

### 根本原因

1. **前端数据加载时序问题**（主要问题）
   - `loadAllData()` 使用 `Promise.all` 并行加载三个异步操作
   - 并行执行导致在合并主题时，`allThemes.value` 可能还未被 `loadStoreThemes()` 更新
   - 代码逻辑：
     ```javascript
     await Promise.all([loadMyThemes(), loadStoreThemes(), loadGamesList()]);
     const themes = [...allThemes.value];  // ❌ 此时可能还是空数组
     ```

2. **后端数据库字段缺失**（次要问题）
   - 实体类 `ThemeInfo` 有 `owner_type` 和 `owner_id` 字段
   - 但数据库表 `theme_info` 可能缺少这两个字段
   - 导致 MyBatis-Plus 分页查询时 `total` 返回 0

### 修复方案

#### 1. 前端：修复数据加载时序
**文件**: `kids-game-frontend/src/modules/creator-center/index.vue`

修改 `loadAllData()` 函数，从并行改为串行执行：

```javascript
// 修改前（并行执行）
async function loadAllData() {
  await Promise.all([
    loadMyThemes(),
    loadStoreThemes(),
    loadGamesList()
  ]);
  // ...
}

// 修改后（串行执行）
async function loadAllData() {
  await loadMyThemes();
  await loadStoreThemes();  // 先加载商店主题
  await loadGamesList();

  // 现在合并时 allThemes.value 已经有数据了
  const themes = [...allThemes.value];
  // ...
}
```

**影响**:
- 确保 `loadStoreThemes()` 先完成并更新 `allThemes.value`
- 避免并行执行导致的竞态条件

#### 2. 后端：修复数据库字段缺失
**文件**: `kids-game-backend/fix-owner-fields.sql`（新建）

创建 SQL 脚本来修复数据库表结构：

```sql
-- 添加 owner_type 和 owner_id 字段
ALTER TABLE theme_info
ADD COLUMN IF NOT EXISTS `owner_type` VARCHAR(20) DEFAULT 'APPLICATION' AFTER `applicable_scope`;

ALTER TABLE theme_info
ADD COLUMN IF NOT EXISTS `owner_id` BIGINT AFTER `owner_type`;

-- 根据 applicable_scope 迁移数据
UPDATE theme_info
SET owner_type = 'APPLICATION'
WHERE applicable_scope = 'all' AND (owner_type IS NULL OR owner_type = '');

UPDATE theme_info
SET owner_type = 'GAME'
WHERE applicable_scope = 'specific' AND (owner_type IS NULL OR owner_type = '');

-- 从 theme_game_relation 表迁移 owner_id
UPDATE theme_info t
INNER JOIN theme_game_relation r ON t.theme_id = r.theme_id
SET t.owner_id = r.game_id
WHERE t.applicable_scope = 'specific' AND (t.owner_id IS NULL OR t.owner_id = 0);
```

**影响**:
- 添加缺失的 `owner_type` 和 `owner_id` 字段
- 迁移现有数据到新字段
- 修复 MyBatis-Plus 分页查询的 `total` 问题

## 主题创作中心筛选功能修复（2026-03-21 20:31）

### 问题
主题创作中心的主题列表展示与筛选条件不匹配，无论选择什么筛选条件，主题列表都是混乱的。

### 根本原因
在 `reloadCurrentData()` 函数中：
1. **加载官方主题时**：正确地根据 `filterOwnerType`、`selectedGameId` 筛选参数调用后端 API
2. **合并我的主题时**：完全没有根据筛选条件过滤，直接把所有"我的主题"都添加到列表中

这导致：
- 选择"官方"或"购买"时，仍然显示所有"我的主题"
- 选择特定游戏时，仍然显示该用户的所有游戏主题
- 主题列表与筛选条件完全不一致

### 修复方案

#### 1. 修复筛选逻辑（creator-center/index.vue）

在 `reloadCurrentData()` 函数中，添加完整的筛选条件：
```typescript
// 只有当筛选条件包含"我的"时，才添加我的主题
if (themeSourceFilter.value === 'all' || themeSourceFilter.value === 'mine') {
  myThemes.value.forEach((theme: any) => {
    // ⭐ 过滤条件 1: 适用范围匹配（ownerType）
    if (filterOwnerType.value && theme.ownerType !== filterOwnerType.value) {
      return; // 跳过不匹配的主题
    }

    // ⭐ 过滤条件 2: 游戏匹配（如果选定了具体游戏）
    if (filterOwnerType.value === 'GAME' && selectedGameId.value && theme.ownerId !== selectedGameId.value) {
      return; // 跳过不匹配的游戏主题
    }

    // ⭐ 避免重复（根据 themeId 去重）
    const exists = themes.find(t => t.themeId === theme.themeId);
    if (!exists) {
      themes.push({ ...theme, source: 'mine', sourceLabel: '我的', sourceIcon: '🎨' });
    }
  });
}
```

#### 2. 清理遗留的 applicableScope 引用（ThemeCreator.vue）

根据之前的决策，`applicableScope` 字段已被废弃，统一使用 `ownerType`。

修复前：
```typescript
const payload = {
  applicableScope: formData.basic.applicableScope, // ❌ 遗留字段
  // ...
};
```

修复后：
```typescript
const payload = {
  ownerType: formData.basic.selectedGameCode ? 'GAME' : 'APPLICATION',
  ownerId: formData.basic.selectedGameId,
  // ...
};
```

### 筛选规则
- **适用范围 (ownerType)**: `APPLICATION` - 应用主题 / `GAME` - 游戏主题
- **主题来源**: `all` - 全部 / `official` - 官方 / `purchased` - 购买 / `mine` - 我的
- **游戏筛选 (selectedGameId)**: 仅在 `ownerType === 'GAME'` 时生效，根据 `ownerId` 筛选

public List<ThemeInfo> getPurchasedThemes(Long buyerId) {
    LambdaQueryWrapper<ThemePurchase> wrapper = new LambdaQueryWrapper<>();
    wrapper.eq(ThemePurchase::getBuyerId, buyerId)
           .eq(ThemePurchase::getIsRefunded, 0);

    List<ThemePurchase> purchases = themePurchaseMapper.selectList(wrapper);

    }

    List<Long> themeIds = purchases.stream()
            .map(ThemePurchase::getThemeId)
            .collect(Collectors.toList());

    themeWrapper.in(ThemeInfo::getThemeId, themeIds);

}

主题管理页面中，已下架的主题只有作者本人才能看到，非本人创作的下架主题应该被过滤掉。

### 实现方案

#### 2. Service 实现修改（ThemeServiceImpl.java）

```java
// 已下架主题过滤：已下架的非本人创作主题不显示
if (authorId != null) {
    // (status != 'offline') OR (status = 'offline' AND authorId = 当前用户)
    wrapper.and(w -> w
        .ne(ThemeInfo::getStatus, "offline")
        .or(w2 -> w2
            .eq(ThemeInfo::getStatus, "offline")
            .eq(ThemeInfo::getAuthorId, authorId)
        )
    );
}
```

#### 3. Controller 修改（ThemeController.java）

```java
// 获取当前登录用户 ID，用于已下架主题过滤
String userIdStr = (String) request.getAttribute("userId");
Long authorId = (userIdStr != null && !userIdStr.isEmpty()) ? Long.valueOf(userIdStr) : null;

Page<ThemeInfo> pageInfo = themeService.listThemes(ownerType, ownerId, status, page, pageSize, authorId);
```

### 过滤逻辑说明

当用户传入 `authorId` 时：
- **上架中/审核中的主题**：所有用户都能看到
- **已下架的主题**：只有 `authorId` 等于该主题作者的用户才能看到

这意味着用户 A 创建的主题被下架后：
- 用户 A 可以看到（因为 `authorId = A`）
- 用户 B 不能看到（因为 `authorId ≠ A`）
