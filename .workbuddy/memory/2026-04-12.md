## 游戏素材生成工具 (game-ui-tool)

在 `game-ui-tool` 目录下创建了基于 sd-webui-aki API 的游戏素材生成工具库。

### 核心文件
- `src/index.ts` - 主模块，封装 SDWebUI 类
- `src/types.ts` - TypeScript 类型定义
- `package.json` - 项目配置
- `README.md` - 使用文档
- `run.ts` - 使用示例

### 主要功能
1. **文生图** (txt2img)
2. **图生图** (img2img)
3. **局部重绘** (inpainting)
4. **10种游戏风格预设**: pixel-art, cartoon, fantasy, scifi, medieval, horror, minimalist, chibi, watercolor, hand-drawn
5. **30+ Prompt模板**: 角色、道具、场景、UI元素、怪物、背景

### 重要：PNG 格式
- **所有生成的素材均为 PNG 格式**，支持透明背景
- 代码中强制设置 `format: 'png'`
- 适合游戏开发使用

### AI 集成场景
当 AI（如 CodeBuddy Agent）进行游戏编码时：
1. AI 编写游戏代码
2. AI 调用 game-ui-tool 生成游戏素材
3. 素材自动保存到项目目录（如 `assets/characters/hero.png`）

## PVZ 游戏全屏优化

**问题**：pvz 游戏存在严重的逻辑分辨率不一致（main.js 设 800×450，BootScene 覆盖为 490×290），导致全屏体验混乱，场景间视觉跳变。

**修复方案**：
1. 统一逻辑分辨率为 980×580（原版 490×290 ×2，全屏友好）
2. Phaser Scale 模式改为 `RESIZE`（自适应窗口大小）
3. 网格布局统一为 12列×7行
4. 移除所有手动 resize hack（各场景的 handleResize、相机 zoom 缩放）
5. index.html 增加 `viewport-fit=cover`、`position:fixed` 防 iOS 弹性滚动、F11 监听

**修改文件**：
- `src/main.js` — Phaser config 统一为 980×580 + RESIZE
- `src/scenes/BootScene.js` — 从 GAME_CONFIG 读取布局常量
- `src/scenes/PlayScene.js` — 全面重写布局（放大字号/元素适配新分辨率）
- `src/scenes/TitleScene.js` — 移除 resize hack
- `src/scenes/OverScene.js` — 移除 resize hack
- `src/ui/SeedCard.js` — 卡片尺寸从 50×58 增大到 58×58
- `index.html` — 全屏样式优化、F11 监听、安全区域适配
