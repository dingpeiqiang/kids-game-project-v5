## rpgShooterTowerDefense 架构大重构（09:00-09:10）

### 根因
init.ts 是 1505 行的"上帝文件"，包含渲染、输入、游戏逻辑全部内联。renderer.ts/input.ts 虽然存在但未被调用，导致改 UI 时改错文件。

### 重构结果

| 文件 | 重构前 | 重构后 | 职责 |
|------|--------|--------|------|
| **init.ts** | **1505行** | **201行** (-87%) | 纯初始化+组装入口 |
| **renderer.ts** | 590行（未调用） | ~420行（实际生效） | 所有渲染：UI/摇杆/弹窗/按钮 |
| **input.ts** | 381行（未调用） | ~380行（实际生效） | 所有输入：触摸/鼠标/键盘/摇杆 |
| **gameLoop.ts** | 不存在 | ~100行（新建） | 每帧逻辑更新 |

### 新增文件
- `gameLoop.ts`：从 init.ts 提取 `updateGame()` 函数
- `renderer.ts`：完全重写，包含 `render()`, `drawJoystick()`, `drawTurretButtons()`, `drawUpgradeDialog()` 等
- `input.ts`：完全重写，`initInput()` 返回 cleanup 函数，`handleMapClick()` 统一处理点击

### 关键设计决策
1. `RenderContext` 接口统一传递渲染所需引用（state/joystick/mobileButtonsRef等）
2. `InputInitParams` 接口统一输入初始化参数
3. 摇杆提示改为醒目青色虚线框+方向箭头+🎮移动标签
