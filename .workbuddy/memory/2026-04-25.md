# 2026-04-25 工作日志

## Docker 日志问题修复

**问题**：服务器上看不到 Docker 启动日志。

**根本原因**：
1. `deploy-from-images.sh` 用 `&` 后台启动日志收集进程，SSH 断开后进程被 SIGHUP 杀死
2. `docker/docker-compose.lowmem.yml` 中 mysql init.sql volume 路径错误（`./docker/mysql/init.sql`），该文件在 `docker/` 目录下应为 `./mysql/init.sql`
3. backend/frontend 的 `build.context` 是 `.`（docker/），但 Dockerfile 路径没更新
4. `fast-deploy.sh` 仍引用旧根目录路径（`.env.production.example`、`docker-compose.lowmem.yml`）

**修复内容**：
- `docker/scripts/deploy-from-images.sh`：日志收集改用 `nohup`
- `docker/scripts/manage-logs.sh`：`restart` 动作改用 `nohup`  
- `docker/docker-compose.lowmem.yml`：
  - mysql volume 路径修复为 `./mysql/init.sql`
  - backend/frontend `build.context` 改为 `..`，`dockerfile` 改为 `docker/Dockerfile.*`
  - 所有服务添加 `logging: json-file` 配置
- `fast-deploy.sh`：
  - `COMPOSE_FILE` 改为 `docker/docker-compose.lowmem.yml`
  - `.env` 检查路径改为 `docker/.env`
  - 兼容 `docker compose`（v2）和 `docker-compose`（v1）

**服务器上直接查看日志的命令**：
```bash
docker logs kids-game-backend --tail=200 -f
docker compose -f docker/docker-compose.lowmem.yml logs -f backend
```

## 前端容器 import.meta.env undefined 修复（2026-04-25）

**方案**：简化版 — 生产环境全走相对路径，Nginx 代理转发，零运行时注入。

**根因**：`import.meta.env` 是 Vite 构建时静态替换，容器运行时环境变量无效。

**简化后的链路**：
```
浏览器 → Nginx(80) → 后端(8080)  同一 Docker 网络，前端不感知后端地址
```

**改动的文件**：
- `src/core/config/env.ts`：production 全用相对路径（`/api`、`/ws`、`/resources`）
- `index.html`：去掉 `env.js` 脚本
- `docker/Dockerfile.frontend`：`CMD nginx`，删掉 entrypoint
- `docker/docker-compose.lowmem.yml`：去掉 frontend environment 配置
- `docker/nginx/frontend.conf`：补全 `/ws/` + `/resources/` 代理

**受影响文件（已改用 envConfig）**：
- `request.ts`、`base-api.service.ts`、`ImageResourcePanel.vue`、`AudioResourcePanel.vue`、`GameContainer.vue`

**服务器部署命令**：
```bash
cd docker
docker compose -f docker-compose.lowmem.yml up -d --build
```
