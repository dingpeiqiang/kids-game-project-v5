# 2026-04-05 工作日志

## space-invaders 子弹卡住/不移动修复（01:01→01:07）

**根本原因**：Phaser 对象池复用子弹时，被 `destroy()` 后的物理体处于禁用状态。`setActive(true)` 不会自动重新启用物理体，导致 `setVelocityY(-520)` 无效，子弹卡在原地不动。

**修复方案**（script-offline.js）：
- `fireBullet` 改用 `b.enableBody() + b.body.reset(x, y) + setActive(true).setVisible(true)` 三步
- `body.reset()` 重置位置并启用物理体；`enableBody()` 确保被禁用过的物理体被重新激活；**`setActive(true)` 是关键**——`enableBody()` 只启用物理体，不改变 sprite 的 active 标志，缺失后 sprite 不参与渲染和 update 循环，子弹出现后即被清理
- ⚠️ Phaser Arcade Sprite 没有 `reset()` 方法，只有 `body.reset()`

## space-invaders 爆炸特效黑框修复（01:09）

**根本原因**：Phaser Graphics 的 `fillCircle` + `generateTexture` 生成爆炸纹理时，图形渲染可能失败（黑色占位 = 纹理未正确绑定），显示为黑框。

**修复方案**：改用原生 Canvas 2D API 生成爆炸纹理（三层同心圆：橙色外圈 + 黄色中圈 + 白色核心），通过 `this.textures.addCanvas('explosion', cv)` 注册。

## space-invaders 子弹残留修复（00:57）

**根本原因**：越界子弹只做 `setActive(false).setVisible(false)`，对象留在渲染层；清理条件 `y < -10` 不够宽；bullet pool `maxSize: 30` 可能耗尽。

**修复方案**（script-offline.js）：
- 子弹组 `maxSize: 30 → 60`，`enemyBullets: 50 → 80`，添加 `runChildUpdate: true`
- 清理时用 `b.destroy()` 直接销毁对象，不留残留
- 清理范围扩大到四边：`(y < -20 || y > 620 || x < -20 || x > 820)`

## space-invaders 敌人飞出屏幕修复（00:54）

**两个根本原因**：
1. `checkEnemyBoundary` 只在 `hitEnemy` 回调末尾调用，不射击时永远不执行边界检测
2. 边界检测触发点 `x > 750` 太靠近边缘，高速时一帧内直接越过，检测逻辑完全跳过

**修复方案**（script-offline.js）：
- 将 `checkEnemyBoundary` 移到 `update` 主循环末尾，每帧必定执行
- 收紧触发点为 `x >= 740 / x <= 60`
- 新增强制钳位：`e.x > 790 → e.x = 790`，越界后强制拉回并触发换向
- 删除 `hitEnemy` 末尾多余的 `checkEnemyBoundary` 调用

## space-invaders 子弹穿透修复（00:52）

**根本原因**：与道具重复提示同一类问题。`hitEnemy`/`hitBoss` 中用 `setActive(false)` 禁用子弹，但同帧内子弹仍在物理世界，可继续触发 overlap 回调，导致一发子弹打穿多个敌人。

**修复方案**（script-offline.js）：
- `hitEnemy`：开头加 `if (!bullet.active || !enemy.active) return`，然后 `bullet.disableBody(true, true)` + `enemy.disableBody(true, true)`（将原来末尾的 `enemy.disableBody` 提前）
- `hitBoss`：同样改为 `bullet.disableBody(true, true)` + 防重入

## space-invaders 道具重复提示修复（00:47）

**根本原因**：Phaser `physics.add.overlap` 回调每帧都会触发，`setActive(false)` 不能立即阻止同一帧内的后续回调，导致吃一个道具显示多次提示/音效。

**修复方案**（script-offline.js）：
- `collectPowerup`：开头加 `if (!powerup.active) return`，然后立即调用 `powerup.disableBody(true, true)` 彻底从物理世界移除
- `playerHitByBullet`：同样加 `if (!bullet.active) return` + `bullet.disableBody(true, true)`
- 关键区别：`disableBody(true, true)` vs `setActive(false)` —— 前者立即从 Arcade 物理步骤中剔除，后者只改标志位下帧生效

## space-invaders 游戏体验优化（00:39-01:00）

**修改的文件**：
- `public/script-offline.js` — 全量重写，修复所有体验问题
- `views/v3.html` — 重写页面布局，修复画布适配

**修复的 Bug / 优化项**：

1. **敌人移动速度异常**：原代码同时使用"1s 定时器步进 + update 每帧移动"，实际速度是预期的数十倍。改为纯速度驱动（`setVelocityX`），边界检测换向逻辑移入 `hitEnemy` 回调后调用 `checkEnemyBoundary()`。

2. **玩家无敌帧缺失**：新增 `invincible` 标志 + 1.5s timeout，受伤期间所有伤害被跳过，防止一次碰撞扣光所有生命。

3. **敌人子弹缺失**：`script-offline.js` 中原本没有 `enemyBullets` 组和 `playerHitByBullet` 碰撞检测，导致敌人子弹穿透玩家。已补齐完整敌人射击系统。

4. **levelComplete 重复触发**：`gameWon` 标志检查放到函数开头，加上防重入 guard；`update` 中也添加 `!gameWon` 前置判断。

5. **AudioContext 自动播放限制**：原代码在脚本顶层直接 `new AudioContext()`，现代浏览器会报 "not allowed to start" 警告并静音。改为延迟初始化 `ensureAudio()`，首次用户交互后才创建。

6. **画布不自适应容器**：Phaser config 加入 `scale: { mode: Phaser.Scale.FIT, autoCenter: CENTER_BOTH }`；v3.html 重构为 flex 全高布局，`#game-container` 撑满剩余空间。

7. **移动端缺乏触屏控制**：`setupTouchControls()` 在触屏设备上自动添加左移/右移/射击虚拟按钮，绑定 pointerdown/up/out 事件。

**额外改进**：
- 相机震动（`cameras.main.shake`）替代全局变量 screenShake 方案
- Boss 血条颜色随 HP 变化（绿→橙→红）
- 连击文字颜色渐变
- GAME OVER 界面新增命中率、最大连击统计
- 难度选择界面重新设计，更美观清晰
