## Bug 修复（22:35）

### 问题：复活后子弹可以穿过玩家
**根本原因**：
- `PlayerCombatManager` 缺少 `reset()` 方法
- 复活时没有重置 `isShieldActive` 状态
- 导致碰撞回调中的护盾检查逻辑异常

**修复内容**：
1. `PlayerCombatManager.ts` 新增 `reset()` 方法
   - 重置 `isShieldActive = false`（关键！）
   - 重置 `currentArmor = 0`
   - 重置 `isFrozen = false`

2. `PlayerCombatManager.ts` 的 `startRespawn()` 方法
   - 复活时调用 `this.reset()` 清空战斗状态

**文件变更**：
- `src/managers/PlayerCombatManager.ts` - 添加 reset/destroy 方法 + 复活时调用 reset

### Bug 修复（22:38）- 敌人子弹穿过玩家

**根本原因**：
- `getState()` 返回 `PlayerState` 枚举（如 `PlayerState.RESPAWNING`）
- 但代码中用 `'RESPAWNING'` 字符串比较，导致比较失败
- 复活后无敌状态检测失败，敌人子弹直接穿过玩家

**修复内容**：
- `CollisionManager.ts` 导入 `PlayerState` 枚举
- 两处比较 `'RESPAWNING'` → `PlayerState.RESPAWNING`

```typescript
// 修复前（错误）
if (stateManager?.getState() === 'RESPAWNING')

// 修复后（正确）
if (stateManager?.getState() === PlayerState.RESPAWNING)
```

**文件变更**：
- `src/managers/CollisionManager.ts` - 导入 PlayerState + 修复 2 处状态比较

### Bug 修复（22:41）- 敌人子弹穿过玩家（最终解决）

**根本原因**：
- 碰撞回调中先检查 `hit` 状态，后检查无敌状态
- 子弹命中后延迟重置 `hit = false`，在重置前子弹被当作"已命中"直接穿过

**最终修复**：
- 调整检查顺序：无敌状态检查优先于 `hit` 检查
- 复活后无敌期间子弹直接穿过玩家

**文件变更**：
- `src/managers/CollisionManager.ts` - 调整 2 处碰撞回调的检查顺序
