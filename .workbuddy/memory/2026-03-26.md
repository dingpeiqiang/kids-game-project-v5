# 2026-03-26 工作日志

## 贪吃蛇道具系统 Bug 修复（两轮）

### 第一轮：碰撞检测不工作、道具不消失
修复了 PhaserGame.update() 传入空数组、坐标系不一致、ItemSystem offsetY 未含 safeTop、重复调用 applyItemEffect 4 个 bug。

### 第二轮：道具效果看不出来
**根本原因**：PhaserGame.gameData 与 gameStore 完全脱节（数据孤岛）
**修复**：game.ts 新增 itemEffects 状态 + applyItemEffect() 统一入口，moveSnake/addScore/碰撞检测全部接入；SnakeGame.vue 新增道具效果状态栏 UI。

### 第四轮：道具效果仍不生效 + 无音效
**错误**：`gameStore.applyItemEffect is not a function`
**原因**：在 Phaser class 内部（非 Vue 上下文）调用 `useGameStore()` 时，Pinia 热更新后返回的 store 实例不包含新方法
**修复**：
1. `PhaserGame.ts` 删除内部的 `useGameStore()` 调用，新增 `onItemEffect` 回调字段 + `setItemEffectCallback()` 方法
2. `SnakeGame.vue` 中创建 `PhaserGame` 后立即调用 `setItemEffectCallback(() => gameStore.applyItemEffect(type))`（两处：首次加载 + 重玩）
3. `handleItemCollected` 调用 `this.playSound('item_collect')`，legacySoundMap 映射到 `effect_levelup`

**核心教训**：Pinia `useGameStore()` 必须在 Vue 组件 setup 上下文中调用，不能在 Phaser class 内部调用

### 第三轮（当日晚）：再次游玩时崩溃
**错误**：`Cannot read properties of undefined (reading 'activeEffects')` @ SnakeGame.vue:74
**原因**：`resetItemEffects()` 用整体赋值 `itemEffects.value = {...}` 导致路由切换过渡动画期间 template 访问到 undefined
**修复**：
1. `resetItemEffects()` 改为逐字段重置（不替换整个对象）
2. template 的 `v-if` 和 `v-for` 加 `?.` 可选链兜底
