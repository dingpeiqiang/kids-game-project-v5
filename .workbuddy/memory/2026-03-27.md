## 全屏屏幕适配修复（参考贪吃蛇方案）

**问题**：test-game 使用 `Phaser.Scale.FIT`，画布不会撑满整个屏幕，只按固定尺寸等比缩放。

**参考**：分析了贪吃蛇的完整适配架构，改为多层协同方案。

**修复方案（4层配合）**：

1. **index.html**: `viewport-fit=cover` + `env(safe-area-inset-*)` padding
2. **App.vue**: `100vw × 100vh` + `overflow: hidden` + 安全区域 padding + fade 动画
3. **GameView.vue**: `h-screen w-full overflow-hidden` + `touch-action: none` + `user-select: none`
4. **Phaser scale**: `mode: RESIZE` + `autoCenter: CENTER_BOTH` + `width: '100%', height: '100%'`

**关键区别**：
- `FIT` = 固定画布等比缩放（不会撑满），`RESIZE` = 画布跟随容器动态变化（撑满）
- RESIZE 模式下，游戏逻辑必须用 `scene.scale.width/height` 获取实时尺寸，不能用固定值
- 贪吃蛇额外有 cellSize 动态计算 + resize 监听重建，简单游戏可不需要
