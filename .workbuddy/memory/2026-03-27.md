## 框架 UI 缩放方案重构：CSS calc → provide/inject（第六轮）

### 问题
框架组件使用 `calc(Npx * var(--ui-scale))` CSS 变量缩放，与贪吃蛇的 `ui.getFontSize(N)` JS 缩放存在精度差异，导致布局、大小整体不一致。

### 根因
CSS `calc()` 和 JS `baseSize * uiScale` 的计算时机不同：
- CSS `--ui-scale` 是 `initUIParams` 时设置的**静态值**，resize 时才更新
- JS `ui.getFontSize()` 是调用时**实时计算**，值更精确
- 两者可能差 0.1-0.5px，累积到布局上就很明显

### 解决方案：provide/inject 桥接
**新增 `composables/useScale.ts`**：
- `provideUIScale(app, scaleFns)` — 游戏项目在 main.ts 注册缩放函数
- `useScale()` — 框架组件内获取缩放函数，返回和 `useResponsiveUI()` 完全一致的方法

**修改的文件**：
1. `kids-game-framework/src/composables/useScale.ts` — **新增** provide/inject 桥接
2. `kids-game-framework/src/index.ts` — 导出 `provideUIScale` / `useScale`
3. `kids-game-framework/src/components/ui/GameButton.vue` — `useScale()` 替代 CSS calc
4. `kids-game-framework/src/components/ui/SoundToggle.vue` — `useScale()` 替代 CSS calc
5. `kids-game-framework/src/components/ui/ThemeSelector.vue` — `useScale()` + `v-bind` 替代 CSS calc
6. `kids-game-framework/template/src/main.ts` — `provideUIScale(app, {...})`
7. `games/test-game/src/main.ts` — `provideUIScale(app, {...})`

### 使用方式（游戏项目 main.ts）
```ts
import { provideUIScale } from '@kids-game/framework'
import { useResponsiveUI } from '@/utils/uiResponsive'
const ui = useResponsiveUI()
provideUIScale(app, {
  getFontSize:     ui.getFontSize.bind(ui),
  getPadding:      ui.getPadding.bind(ui),
  getGap:          ui.getGap.bind(ui),
  getWidth:        ui.getWidth.bind(ui),
  getHeight:       ui.getHeight.bind(ui),
  getBorderRadius: ui.getBorderRadius.bind(ui),
  uiScale:         ui.uiScale
})
```

## 修复 App.vue 布局横排 Bug（第五轮）

### 问题
test-game 首页内容横着排列（所有元素在一行显示）。

### 根因
`App.vue` 的 `.app-container > *` 全局规则缺少 `flex-direction: column`：
```css
/* 修复前 — 默认 flex-direction: row 导致子元素横排 */
.app-container > * {
  flex: 1;
  display: flex;          /* ← 默认是 row！ */
  align-items: center;
  justify-content: center;
}
```
`<transition>` 包裹的视图组件被这条规则匹配，`display: flex` 默认 row 方向排列。

### 修复
在 `.app-container > *` 中补 `flex-direction: column`。

## 全屏屏幕适配修复（参考贪吃蛇方案）

**问题**：test-game 使用 `Phaser.Scale.FIT`，画布不会撑满整个屏幕，只按固定尺寸等比缩放。

**参考**：分析了贪吃蛇的完整适配架构，改为多层协同方案。

**修复方案（4层配合）**：

1. **index.html**: `viewport-fit=cover` + `env(safe-area-inset-*)` padding
2. **App.vue**: `100vw × 100vh` + `overflow: hidden` + 安全区域 padding + fade 动画
3. **GameView.vue**: `h-screen w-full overflow-hidden` + `touch-action: none` + `user-select: none`
4. **Phaser scale**: `mode: RESIZE` + `autoCenter: CENTER_BOTH` + `width: '100%', height: '100%'`

**关键区别**：
- `FIT` = 固定画布等比缩放（不会撑满），`RESIZE` = 画布跟随容器动态变化（撑满）
- RESIZE 模式下，游戏逻辑必须用 `scene.scale.width/height` 获取实时尺寸，不能用固定值
- 贪吃蛇额外有 cellSize 动态计算 + resize 监听重建，简单游戏可不需要

**修改文件**：
- `games/test-game/index.html`, `App.vue`, `GameView.vue`, `game/index.ts`
- `kids-game-framework/template/index.html`, `App.vue`, `GameView.vue`, `game/index.ts`
- `kids-game-framework/src/core/GameEngine.ts`（默认 scaleMode 改为 RESIZE）
