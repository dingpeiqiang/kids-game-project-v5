# 2026-03-28 工作日志

## 配置参数未生效 bug 修复（2026-03-28）

**问题根因**：`GameSettingsPanel` 保存的 `currentGameConfig` 通过 `sessionStorage` 中转，但 `ComponentGameScene` 只读了 `difficulty/enableDynamicDifficulty`，而 `SnakePhaserGame`（`PhaserGame.ts`）完全忽略 sessionStorage，导致 `speed/initialLength/cellSize/分数` 全部无效。

**修复链路**：
1. **`types/game.ts`** — `DifficultyConfig` 新增 `initialLength` 字段，`DIFFICULTY_CONFIGS` 各难度补充 `initialLength`
2. **`stores/game.ts`** — 新增 `CustomGameConfig` 类型 + `customConfig: ref<CustomGameConfig | null>` + `setCustomConfig()` 方法；`startGameWithInit()` 优先读 `customConfig.initialLength`；`moveSnake()` 优先读 `customConfig.speed`；`generateFood()` 优先读 `customConfig.*FoodScore`
3. **`DifficultyView.vue`** — `startGame()` 改为：① `gameStore.setDifficulty(selectedDifficulty)` ② `gameStore.setCustomConfig(currentGameConfig)` ③ 不再走 sessionStorage；`selectedDifficulty` 初始值从 `gameStore.difficulty` 同步；去掉多余的 `gameStore.startGame()` 调用
4. **`SnakeGame.vue`** — 初始化时用 `startGameWithInit(cellSize)` 替代 `resetGame+startGame+generateFood`；`cellSize` 优先读 `customConfig.cellSize`（否则用 Phaser 自动计算值）

**生效的参数**：蛇初始长度、移动速度、格子大小、普通/奖励/特殊食物分数、各难度预设 initialLength

## uiResponsive.ts 响应式重构

**问题根因**：原来 `uiScale` 是模块级普通变量，`useResponsiveUI()` 返回的是值快照，resize 后 Vue computed 不会重新触发。

**重构方案**：
- `_screenW`/`_screenH` 改为 `ref()`，`uiScaleRef` 改为 `computed()`（模块级单例，全局共享）
- 所有 `getFontSize()`/`getPadding()` 等工具函数内部读取 `uiScaleRef.value`，在组件 `computed` 内调用时自动建立响应式依赖
- 全局 resize 监听统一在 `_ensureResizeListener()` 中注册（只注册一次），各视图无需再手动 `addEventListener/removeEventListener`

**修改文件**：
- `src/utils/uiResponsive.ts` — 核心重构
- `src/views/StartView.vue` — 移除 handleResize + addEventListener/removeEventListener
- `src/views/GameOverView.vue` — 修正 uiScale.value
- `src/views/DifficultyView.vue` — 移除冗余 uiScale computed，传 .value 给子组件
- `src/components/game/SnakeGame.vue` — 移除手动 resize 监听

**关键 API 变化**：
- `ui.uiScale` 原来是 number，现在是 `ComputedRef<number>`，模板中直接用 `ui.uiScale.value` 或在模板里 Vue 自动解包



**问题**：`DifficultyView.vue` 传给 `GameButton` 的 `fontSize`/`paddingLeft` 等 props 是 `ui.getFontSize()` 返回的带 `px` 字符串，但 `GameButton` 的 props 声明为 `number`（原始设计尺寸），内部自行缩放。导致 Vue warn。

**修复**：改为传字面量数字（如 `:fontSize="26"`），删除多余的 `btnFontSize`/`btnPaddingH`/`btnPaddingV` computed。
