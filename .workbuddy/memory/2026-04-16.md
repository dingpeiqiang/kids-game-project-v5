# 2026-04-16 工作日志

## pvz 游戏 sprites 图集加载修复

**问题**：Phaser 报错 `Texture "sprites" has no frame "__BASE"`，导致 plant.js 的 shoot 方法崩溃（无法创建子弹）。

**根因**：BootScene.js 用 `this.load.multiatlas()` 加载 sprites.json，但该文件是 JSONHash 格式（`frames` 为对象），而 `multiatlas()` 期望 TexturePacker 的 multi-atlas 格式（含 `textures` 数组和 `__BASE` 帧）。

**修复**：BootScene.js 中将 `multiatlas()` 改为 `atlas(key, image, json)` —— atlas() 兼容 JSONHash 格式。

## pvz 豌豆子弹纹理修复

**问题**：射出的豌豆图片资源不对，sprites 图集里的 `pea.png` 帧是 160×160 大图，不是小圆球。

**根因**：plant.js / iceshooter.js / pea.js 全部使用 `sprites` 图集帧（160×160），但 assets 目录下已有独立的 `pea.png`（15×15 小圆球）和 `ice_pea.png`（15×15 蓝色小球）。

**修复**：
1. GTRS.json images.scene 中注册 `pea` 和 `ice_pea` 为独立图片资源
2. plant.js `shoot()` 改用纹理 `'pea'`，setScale(2) → 30px 显示，body.setSize(15,15)
3. iceshooter.js `shoot()` 改用纹理 `'ice_pea'`，同样 setScale(2)
4. pea.js 构造函数改用纹理 `'pea'`，去掉 sprites 图集帧依赖
