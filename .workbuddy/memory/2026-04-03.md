## 敌人坦克"倒着往下移动"问题修复（00:40）

**问题描述**：
- 敌人坦克从屏幕上方向下移动时，炮口朝上
- 看起来像"倒着走"，与经典坦克大战游戏行为不一致

**根本原因**：
**经典坦克大战游戏资源的图片命名与实际朝向相反**

| 文件名 | 实际图片中的炮口朝向 |
|--------|-------------------|
| `enemy_light_up.png` | ⬇️ **向下** |
| `enemy_light_down.png` | ⬆️ **向上** |
| `enemy_light_left.png` | ➡️ **向右** |
| `enemy_light_right.png` | ⬅️ **向左** |

这是经典坦克大战游戏的标准设计，原因是图片需要旋转才能正确显示。

**修复方案**：

1. **TankSpawner.ts（setupEnemyAI 方法）**：
   ```typescript
   // 向下移动时使用向上纹理（图片中炮口朝下），旋转 90°
   enemy.body.setVelocity(0, enemy.speed)
   enemy.setAngle(90)

   if (enemy.enemyType === 'ENEMY_LIGHT') {
     enemy.setTexture('enemy_light_up')
   } else if (enemy.enemyType === 'ENEMY_MEDIUM') {
     enemy.setTexture('enemy_medium_up')
   } else if (enemy.enemyType === 'ENEMY_HEAVY') {
     enemy.setTexture('enemy_heavy_up')
   }
   ```

2. **EnemyAIManager.ts（updateEnemyDirection 方法）**：
   ```typescript
   // 向上移动：使用向下纹理，旋转 -90°
   if (vy < 0) {
     enemy.setAngle(-90)
     enemy.setTexture('enemy_light_down')
   }
   // 向下移动：使用向上纹理，旋转 90°
   else if (vy > 0) {
     enemy.setAngle(90)
     enemy.setTexture('enemy_light_up')
   }
   // 向左移动：使用向右纹理，旋转 180°
   else if (vx < 0) {
     enemy.setAngle(180)
     enemy.setTexture('enemy_light_right')
   }
   // 向右移动：使用向左纹理，不旋转
   else if (vx > 0) {
     enemy.setAngle(0)
     enemy.setTexture('enemy_light_left')
   }
   ```

**修复效果**：
- ✅ 敌人初始向下移动时，炮口朝下（不再"倒着走"）
- ✅ 所有方向移动时，炮口始终与移动方向一致
- ✅ 完全符合经典坦克大战游戏的行为

## 敌人坦克子弹方向不匹配修复（00:52）

**问题描述**：
- 敌人坦克发射的子弹方向与炮口方向不一致
- 子弹追踪玩家，而不是沿炮口方向发射

**根本原因**：
`enemyShoot()` 方法使用了错误的射击逻辑：
- ❌ 计算敌人到玩家的角度：`Phaser.Math.Angle.Between(enemy.x, enemy.y, player.x, player.y)`
- ❌ 子弹朝玩家方向发射（追踪玩家）
- ❌ 与炮口方向无关

**经典坦克大战正确行为**：
✅ 子弹应该沿着炮口方向发射，而不是追踪玩家

**修复方案**：

修改 `enemyShoot()` 方法，根据敌人的当前速度方向确定子弹方向：

```typescript
// 获取敌人的当前速度
const velocity = enemy.body.velocity

// 根据速度方向确定子弹方向
if (velocity.y < 0) {      // 敌人向上移动
  vx = 0; vy = -bulletSpeed
} else if (velocity.y > 0) { // 敌人向下移动
  vx = 0; vy = bulletSpeed
} else if (velocity.x < 0) { // 敌人向左移动
  vx = -bulletSpeed; vy = 0
} else if (velocity.x > 0) { // 敌人向右移动
  vx = bulletSpeed; vy = 0
}
```

**修复效果**：
- ✅ 敌人向上移动时，子弹向上发射
- ✅ 敌人向下移动时，子弹向下发射
- ✅ 敌人向左移动时，子弹向左发射
- ✅ 敌人向右移动时，子弹向右发射
- ✅ 子弹方向始终与炮口方向一致
- ✅ 符合经典坦克大战游戏行为

## 敌人坦克边界限制修复（00:56）

**问题描述**：
- 敌人坦克可以移出地图边界
- 当所有方向都误判为危险时，敌人会继续移动并最终移出地图

**根本原因**：
- 虽然 `EnemyAIManager` 中有 `isNearBoundary()` 方法检测接近边界并改变方向
- 但缺少强制边界限制逻辑（类似玩家坦克的 `checkBoundaries()`）
- 没有被动保障机制，方向改变失败时敌人会越界

**修复方案**：

1. **新增 `clampToBoundary()` 方法**：
   ```typescript
   private clampToBoundary(enemy: any): void {
     const scene = this.scene as any
     const mapWidth = scene.gridCols * scene.cellSize
     const mapHeight = scene.gridRows * scene.cellSize
     const boundaryPadding = 10
     const halfSize = enemy.displayWidth / 2 || 32

     const minX = boundaryPadding + halfSize
     const maxX = mapWidth - boundaryPadding - halfSize
     const minY = boundaryPadding + halfSize
     const maxY = mapHeight - boundaryPadding - halfSize

     // 强制限制在边界内
     enemy.x = Phaser.Math.Clamp(enemy.x, minX, maxX)
     enemy.y = Phaser.Math.Clamp(enemy.y, minY, maxY)
   }
   ```

2. **在 `updateEnemyAI()` 中强制边界检查**：
   ```typescript
   updateEnemyAI(enemy: any): void {
     if (!enemy || !enemy.active) return

     if (!enemy.body) {
       console.warn('⚠️ [EnemyAI] 敌人没有物理 body，跳过 AI 更新:', enemy)
       return
     }

     // 🔒 强制边界限制（防止敌人移出地图）
     this.clampToBoundary(enemy)  // ⭐ 新增

     // ... 后续逻辑
   }
   ```

**修复效果**：
- ✅ 敌人永远不会移出地图边界
- ✅ 主动预防（`isNearBoundary()`）+ 被动保障（`clampToBoundary()`）双层保护
- ✅ 参考玩家坦克的边界限制逻辑，行为一致
- ✅ 调试日志显示位置修正情况，方便排查问题

**边界参数**：
- `boundaryPadding = 10`: 10px 边界内边距
- 考虑坦克宽度 (`enemy.displayWidth / 2`)
- 使用 `Phaser.Math.Clamp()` 确保位置在合理范围内

## 护盾消耗后玩家仍消失 Bug 修复（第二轮，09:14）

**症状**：护盾消耗后（视觉效果消失），玩家也跟着消失，不复活。

### Bug 9：`handleDeath` 无护盾守卫
死亡流程没有任何护盾还在的检查，若因其他 Bug 误进死亡流程，会直接执行。

### `PlayerCombatManager.ts`
- `consumeShield()`：加二次确认 `isShieldActive`，防止重复调用

- `onHit()`：**新增完整护盾检查**（原来直接透传，现在独立检查 `isShieldActive` 和 `isInvincible`）
- `handleDeath()`：**新增护盾守卫** — 若 `isShieldActive === true` 却进了死亡流程，阻止执行并报错

### `CollisionManager.ts`

## 护盾消耗后玩家仍然消失 Bug 修复（第三轮，09:27）

**症状**：护盾被子弹正确消耗（日志显示"✅ [consumeShield] 护盾已消耗"），但玩家随后仍然消失。

**根本原因**：护盾消耗后（`isShieldActive = false`），**没有任何无敌保护时间**。护盾消失的同一帧或下一帧，敌人子弹/敌人本体再次碰撞玩家 → 直接死亡。

**另一个问题**：`setupEnemyBulletVsPlayer()`（overlap）和 `rebindEnemyBulletVsPlayer()` 的回调里没有护盾检查，子弹命中时直接调用 `onHitWithBullet()`（内部会处理护盾，但不如提前拦截高效）。

**修复内容**：

### `PlayerCombatManager.ts` - `consumeShield()`
- 护盾消耗后调用 `stateManager.startInvincibleTemporary(1000)` 激活**1秒临时无敌**，防止护盾消失后立即被击中死亡
- 添加日志：`🛡️ [PlayerStateManager] 临时无敌激活，持续 1000ms`

### `PlayerStateManager.ts` - 新增 `startInvincibleTemporary(duration)`
- 激活后 `isInvincibleInternal = true`（`isInvincible()` 返回 true）
- 启动闪烁效果（alpha 在 1 和 0.4 之间切换）
- 持续指定时间后自动恢复（清理 timer、设 alpha=1、`isInvincibleInternal = false`）
- 保护 `currentState` 保持 ALIVE（玩家可移动射击）

### `CollisionManager.ts`
- `setupEnemyBulletVsPlayer()` 和 `rebindEnemyBulletVsPlayer()`：在无敌检查之后、重复命中检查之前，增加护盾检查，提前拦截并消耗护盾，避免不必要的函数调用

**修改文件**：
- `src/managers/PlayerCombatManager.ts`
- `src/managers/PlayerStateManager.ts`
- `src/managers/CollisionManager.ts`

**根本原因（4个 Bug 叠加）**：

1. **`pulseEffect` 直接改 `tank.alpha`**：`repeat: -1` tween 把 alpha 持续设为 0.6，与 `PlayerStateManager` 的 `onHit()` 设 alpha=0 和复活闪烁 tween 冲突，导致 alpha 被锁死不可见。

2. **护盾消耗后未清除持续视觉效果**：`consumeShield()` 只设置 `isShieldActive=false`，但 `PowerUpEffectApplier` 里的护盾光圈 Graphics + 颜色脉冲 timer 没有停止，继续干扰坦克状态。

3. **死亡时未清除道具视觉效果**：`handleDeath()` 进入复活流程时，道具的 `repeat:-1` tween 仍在运行，与复活闪烁互相打架。

4. **`clearEffect` 未清除 tint 残留**：停止 timer 后坦克可能卡在某个 tint 颜色上。

**修复内容**：

### `PowerUpEffectApplier.ts`
- `pulseEffect()` 完全重写：**改用 timer 切换 tint，绝不动 alpha**（alpha 由 PlayerStateManager 专属管理）
- 新增 `removeEffectByType(type, sprite)` 方法：按道具类型精准清除视觉效果
- `removeVisualEffects()` 保留，用于全量清除

### `PlayerCombatManager.ts`
- `consumeShield()`：护盾消耗后调用 `powerUpEffectApplier.removeEffectByType('shield')` + `clearTint()`
- `handleDeath()`：进入死亡流程前先 `removeVisualEffects()` 清除所有道具效果
- `startRespawn()`：复活前额外调用 `removeVisualEffects()` 双重保险

**修改的文件**：
- `src/utils/PowerUpEffectApplier.ts`
- `src/managers/PlayerCombatManager.ts`
