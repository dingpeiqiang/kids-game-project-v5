## 坦克大战资源加载系统重构（严格模式）

### 背景
用户要求：游戏资源加载存在问题，不允许使用兜底方案，需重构并严格保障资源加载。

### 核心问题
1. **ResourceManager 架构缺陷**：
   - 在 `create()` 阶段加载资源，与 Phaser 生命周期不符
   - 存在大量兜底逻辑（required=false，失败后继续游戏）
   - 超时和重试机制复杂但不可靠

2. **Orchestrator 问题**：
   - 阶段2与 preload 阶段重复加载
   - 阶段5设置30秒超时兜底
   - 失败时仅警告，不抛出错误

3. **TankGameScene 问题**：
   - playSound() 中音效不存在时静默失败
   - baseDestroyed() 使用 setTint/fallback
   - 缺少严格的资源存在性检查

### 重构内容

#### 1. GameScene.ts - preloadFromGTRS() 严格化
- GTRS 加载失败直接 `throw Error`
- GTRS 结构缺失直接 `throw Error`
- 资源配置缺失 src 直接 `throw Error`
- 加载错误直接 `throw Error`

#### 2. TankGameScene.ts - 新增 preload() 方法
- 在 preload 阶段调用 `preloadFromGTRS()`
- 失败时直接向上抛出，阻断游戏启动

#### 3. TankGameOrchestrator.ts - 阶段2重构
- 从"资源加载"改为"资源验证"
- 移除 ResourceManager 依赖
- 仅验证 `textures.exists()` 和 `cache.audio.exists()`
- 任何缺失直接 `throw Error`

#### 4. TankGameOrchestrator.ts - 阶段5移除超时兜底
- 移除30秒自动完成机制
- 检查 `onLevelComplete` 回调是否存在
- 不存在直接 `throw Error`

#### 5. TankGameScene.ts - playSound() 严格化
- 音效不存在直接 `throw Error`
- 移除 try-catch 兜底
- 移除 `this.sound.get()` 容错检查

#### 6. TankGameScene.ts - baseDestroyed() 严格化
- base 不存在直接 `throw Error`
- `base_destroyed` 纹理不存在直接 `throw Error`
- 移除 setTint/fallback 逻辑

### 技术细节
- 资源加载统一在 preload 阶段完成（符合 Phaser 生命周期）
- 使用 XMLHttpRequest 同步加载 GTRS.json

- 严格验证三层资源映射：GTRS.json → tank_level_1.json → 代码 key

## BGM 文件扩展名修复（20:49）

### 问题描述
运行游戏时出现错误：
```
❌ [音乐资源缺失] bgm_main 未加载
```

### 根本原因
- GTRS.json 中 `bgm_main` 指向 `.mp3` 文件
- 实际生成的是 `.wav` 文件（由 `generate-audio.js` 生成）
- Phaser 无法找到文件，导致加载失败

### 修复方案
修改 `GTRS.json` 中的文件路径：
```json
"bgm_main": {
  "alias": "背景音乐",
  "src": "/themes/tank_default/assets/audio/bgm_main.wav",  // 从 .mp3 改为 .wav
  "type": "wav"  // 从 mp3 改为 wav
}
```

### 修改文件
- `kids-game-house/games/tank-battle/public/themes/tank_default/GTRS.json`

### 状态
✅ 已修复，游戏可以正常加载所有资源

## 敌人坦克物理启用修复（20:54）

### 问题描述
运行游戏时出现错误：
```
TypeError: Cannot read properties of null (reading 'setCollideWorldBounds')
    at EntityManager.createEnemy (EntityManager.ts:392:16)
```

### 根本原因
**核心问题**：`RenderManager.createSprite()` 创建的是普通游戏对象，没有物理体（`body` 属性为 `null`）

**详细分析**：
1. `EntityManager.createEnemy()` 调用 `renderManager.createSprite()` 创建敌人 Sprite
2. `createSprite()` 返回 `Phaser.GameObjects.Sprite`（普通游戏对象）
3. 普通游戏对象**没有** `body` 属性（物理体）
4. 代码尝试调用 `enemy.body.setCollideWorldBounds(true)` 时出错

### 修复方案
在 `EntityManager.createEnemy()` 中，创建 Sprite 后立即启用物理：

```typescript
// 修复前
const enemy: any = this.renderManager.createSprite(x, y, finalTexture, undefined, 'entities')
enemy.body.setCollideWorldBounds(true)  // ❌ enemy.body 是 null

// 修复后
const enemySprite = this.renderManager.createSprite(x, y, finalTexture, undefined, 'entities')
const enemy = this.scene.physics.add.existing(enemySprite, true)  // ✅ 启用物理
console.log(`✅ [EntityManager] 敌人物理已启用: enemy.body=${enemy.body ? '✅' : '❌'}`)
enemy.body.setCollideWorldBounds(true)  // ✅ 现在可以正常调用
```

**关键点**：
- `physics.add.existing(sprite, true)` 将普通 Sprite 转换为物理对象
- 第二个参数 `true` 表示创建动态物体（会受到物理影响）
- 转换后，`enemy.body` 属性才存在

### 其他实体对比
| 实体类型 | 创建方法 | 物理启用方式 | 状态 |
|---------|---------|------------|------|
| 玩家坦克 | `physics.add.sprite()` | 自动启用 | ✅ 正确 |
| 敌人坦克 | `renderManager.createSprite()` | 需要手动启用 | ✅ 已修复 |
| 基地 | `physics.add.sprite()` | 自动启用 | ✅ 正确 |
| 子弹 | `renderManager.createSprite()` + `physics.add.existing()` | 手动启用 | ✅ 正确 |
| 道具 | `powerUpGroup.create()` | 自动启用 | ✅ 正确 |

### 修改文件
- `kids-game-house/games/tank-battle/src/managers/EntityManager.ts`（第 388-392 行）

### 状态
✅ 已修复，游戏可以正常创建敌人并启用物理
