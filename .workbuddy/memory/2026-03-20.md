## 贪吃蛇游戏 - 音频 416 错误修复（第三轮）

### 问题
用户报告 HTTP 416 (Requested Range Not Satisfiable) 错误，加载 `gameover.mp3` 失败。

### 根本原因
1. **所有音频和图片文件都是 0 字节** - 资源生成脚本输出到错误目录
   - 脚本输出到：`public/assets/themes/snake/audio/` 和 `public/assets/themes/snake/scene/`
   - 游戏加载路径：`public/games/snake-vue3/themes/default/audio/` 和 `public/games/snake-vue3/themes/default/images/scene/`
2. **StartView.vue 中的 `buildGTRSThemeJson` 是旧遗留代码**，生成的"假 GTRS"和真正从后端获取的 GTRS 格式不兼容

### 修复内容
1. **修复资源文件**
   - 运行 `generate-resources.mjs` 重新生成资源
   - 复制 WAV 音频到正确目录并创建 MP3 副本（复制 WAV 内容，扩展名为 .mp3）
   - 复制 PNG 图片到正确目录

2. **删除旧代码**
   - 删除 `StartView.vue` 中的 `buildGTRSThemeJson` 函数（约 140 行）
   - 移除 `validateGTRSTheme` 导入（不再需要）
   - 简化校验逻辑：直接在 `themeStore.gtrsRawJson` 中使用已校验的 GTRS 数据

3. **theme.ts 增强**
   - 新增 `gtrsRawJson` ref，存储从后端获取并校验通过的原始 GTRS JSON 字符串
   - 在 `loadThemeFromBackend` 成功后设置 `gtrsRawJson.value = configJsonStr`
   - 导出 `gtrsRawJson` 供 `StartView.vue` 使用

### GTRS 加载流程（严格模式）
1. `themeStore.init()` → 调用 `loadThemeFromBackend()`
2. 从后端 `/api/theme/download?id=xxx` 获取 configJson
3. `validateGTRSTheme()` 严格校验（Ajv Schema）
4. 校验通过 → 解析 JSON → 提取颜色到 `customTheme` → 存储原始 JSON 到 `gtrsRawJson`
5. `StartView.startGame()` → 直接使用 `themeStore.gtrsRawJson`（无需二次校验）
6. `PhaserGame.start()` → 再次从后端加载主题 → 校验 → `applyGTRS()`

### 路径规范化
`normalizeSrcPaths()` 函数自动处理 `/public/` 前缀：
- `/public/games/...` → `/games/...`
