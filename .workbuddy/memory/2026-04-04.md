# 2026-04-04 日志

**问题**：玩家拾取护盾道具后被击中，导致玩家消失且不复活。

**根因分析（3处Bug）**：

### Bug 1 — `PlayerCombatManager.onHit()` 冗余调用死亡路径
- **旧代码**：`onHit()` 最后一行调用 `this.onHitWithBullet()`，导致无护盾时走了两次完整受击流程（扣甲→死亡双触发）
- **修复**：`onHit()` 直接内联护甲扣减和死亡逻辑，不再调用 `onHitWithBullet()`

### Bug 2 — 护盾消耗后 50ms 无敌空窗口
- **旧代码**：`consumeShield()` 用 `delayedCall(50, startInvincibleTemporary)` 延迟激活无敌
- **修复**：改为立即调用 `startInvincibleTemporary(1000)`，消除空窗口

### Bug 3 — `TankGameScene.handlePlayerEnemyCollision` 双重判断
- **旧代码**：Scene 层先判断护盾/无敌，再调用 `combatManager.onHit()`；`onHit()` 内部再次走受击流程
- **修复**：Scene 层去掉所有护盾/无敌判断，统一只调用 `combatManager.onHit()`，由 combatManager 内部负责全部逻辑

### Bug 4 — `PlayerStateManager.startRespawning` respawnCallback 触发时机错误
- **旧代码**：`respawnCallback`（重置玩家位置）在闪烁结束 1.5s 后才调用，玩家在旧坐标停留期间可被再次碰撞
- **修复**：`startRespawning` 立即同步调用 `onRespawnComplete()`，再开始闪烁；移除 `respawnCallback` 字段和 `finishRespawning` 中的相关调用

**修改的文件**：
- `kids-game-house/games/tank-battle/src/managers/PlayerCombatManager.ts`
- `kids-game-house/games/tank-battle/src/managers/PlayerStateManager.ts`
- `kids-game-house/games/tank-battle/src/scenes/TankGameScene.ts`

## 坦克大战：护盾受击后玩家消失（第二轮修复）（07:28）

**问题**：护盾消耗后玩家消失，但护盾光圈还在。

**根因（3处）**：

### Bug A — `consumeShield` 中 `player.active` 判断提前 return
- **旧代码**：`bullet.destroy()` 后立即判断 `if (!player || !player.active) return`
  - 在某些状态（DYING/RESPAWNING）下玩家 `active` 可能是 false
  - 导致 `player.setVisible(true)` / `setAlpha(1)` / `clearTint()` / 护盾视觉清除 / 临时无敌激活全部跳过
  - 结果：玩家不可见 + 护盾光圈残留 + 无敌未激活
- **修复**：
  1. 去掉提前 return，改为强制 `player.setActive(true)` + `setVisible(true)` + `setAlpha(1)`
  2. 护盾视觉清除操作移到 `bullet.destroy()` 之前执行（确保无论如何都清除）
  3. `startInvincibleTemporary` 不再因 `player.active=false` 而跳过

### Bug B — `startInvincibleTemporary` 遇 `player.active=false` 直接 return
- **旧代码**：`if (!player || !player.active) return`（无敌标志未设置）
- **修复**：先设置 `isInvincibleInternal = true`，再强制 `setActive(true) / setVisible(true) / setAlpha(1)`，不提前 return

### Bug C — 护盾光圈 Graphics followTimer 停止后 Graphics 对象未销毁
- **旧代码**：`followTimer` 检测到 `tank.active=false` 时只 `remove()` timer，不销毁 Graphics
- **修复**：同时 `tween.stop()` + `g.destroy()`，确保光圈彻底消失

**修改的文件（第二轮）**：
- `PlayerCombatManager.ts` - `consumeShield()` 重写
- `PlayerStateManager.ts` - `startInvincibleTemporary()` 去除 active 判断
- `PowerUpEffectApplier.ts` - `showShieldOrbit()` followTimer 停止时销毁 Graphics

## 坦克大战：复活后玩家不可见/无法移动修复（10:24）

**问题**：玩家复活后，看不到玩家，且无法移动（速度N/A），但可以射击。

**根因（4处）**：

### Bug 1 — 闪烁效果初始 alpha=0
- **旧代码**：`startBlinkEffect()` 初始 `player.setAlpha(0)`，玩家完全不可见
- **修复**：初始 `player.setAlpha(1)`，反转闪烁逻辑（大部分时间可见 alpha=1，短暂半透明 alpha=0.3）

### Bug 2 — 闪烁效果被意外覆盖
- **旧代码**：`consumeInvincible()` 调用 `setPlayerVisible(true)` 会覆盖闪烁的 alpha
- **修复**：闪烁期间不调用 `setPlayerVisible()`，让闪烁效果继续

### Bug 3 — 闪烁结束后没有正确清理
- **旧代码**：`finishRespawning()` 调用 `cleanupTimers()` 但闪烁回调可能继续执行
- **修复**：立即 `remove(false)` 闪烁定时器，并强制设置 `alpha=1`

### Bug 4 — player 引用未同步到 MovementManager
- **旧代码**：复活后 MovementManager 持有的 player 引用可能丢失
- **修复**：复活后同步调用 `movementManager.setPlayer(player)`

**配置调整**：
- `blinkInterval`: 150ms → 100ms（加快闪烁）
- `blinkCount`: 10 → 6（减少闪烁次数，总时长 600ms）

**修改的文件**：
- `kids-game-house/games/tank-battle/src/managers/PlayerController.ts`
  - `stateConfig`: 闪烁参数调整
  - `startBlinkEffect()`: 反转闪烁逻辑
  - `finishRespawning()`: 立即停止闪烁
  - `consumeInvincible()`: 保护闪烁
  - `activateTemporaryInvincible()`: 保护闪烁
  - `respawn()`: 同步 player 引用
