# 2026-04-10 工作日志

## pvz 游戏修复：阳光无法拾取 + 卡片栏遮挡草地（含二次修复）

### 问题1：阳光无法拾取
**根因**：
1. `sun.js` 重复注册了两次 `pointerdown` 事件（constructor 内写了两次）
2. 点击阳光时，`collect()` 内先开始飞行 tween 后又调用 `killTweensOf`，导致飞行动画被立即取消，`addSun` 回调永远不执行
3. 缺少 `event.stopPropagation()`，点击阳光同时触发了场景的全局 pointerdown（`handleTap`）

**修复**（`src/models/sun.js`）：
- 只注册一次 `pointerdown` 事件
- `collect()` 内先 `killTweensOf` 再创建飞行 tween
- `pointerdown` 回调加入 `event.stopPropagation()`

### 问题2：卡片栏遮挡草地
**根因**：
- `SeedCard` 高度 75px，以 y=35 为中心 → 下边缘到 y=72.5，超出 65px 草地起始线
- 遮挡了草地最上方的格子

**修复**（`src/ui/SeedCard.js` + `src/scenes/PlayScene.js`）：
- 卡片高度从 75 缩为 58px（下边缘 y=60，不超 65）
- 卡片中心 y 从 35 改为 31
- 卡片栏背景 rectangle 高度对应更新为 62px，y 中心改为 31
- SeedCard `pointerdown` 同样加 `event.stopPropagation()` 防止误触种植

### 问题3：背景地图划分不合理（三次修复）
**根因**：
- `BootScene` 中 `CELL_HEIGHT = 290/7 ≈ 41.4px`，基于整个画布高度计算
- 导致第0行格子 y=0~41.4，与卡片栏（y=0~65）完全重叠
- 草地图片被 `setDisplaySize(W, H)` 拉伸铺满整个画布（包含卡片栏区域）

**修复**（`BootScene.js` + `PlayScene.js`）：
- 新增布局常量：`UI_TOP_H=65`（卡片栏高度）、`GAME_TOP_Y=65`（草地起始y）、`GAME_H=225`（草地区域高度）
- `CELL_HEIGHT = GAME_H / 7 ≈ 32.1px`（只基于草地区域）
- 草地图片改为从 `y=GAME_TOP_Y` 开始，`setDisplaySize(490, 225)`，只铺草地区域
- 卡片栏底色为独立 rectangle（depth=98），覆盖顶部 65px
- 所有坐标互转方法（`rowForY/yForRow/getCellOrigin`）加上 `GAME_TOP_Y` 偏移
- 三层布局完全不重叠：卡片栏(y:0~65) / 草地(y:65~290)
