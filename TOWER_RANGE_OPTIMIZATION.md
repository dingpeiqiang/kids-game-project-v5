# 塔防游戏炮台攻击范围优化

## 问题描述
在simple-game的塔防游戏中，部分炮台的攻击范围太小，导致玩家需要密集摆放炮台才能有效防御，游戏体验不佳。

## 优化方案

### 1. RPG Shooter 塔防融合版 (rpgShooterTowerDefense)

#### 激光炮台 (Laser)
- **Lv1**: 130 → **180** (+50, +38%)
- **Lv2**: 新增 → **210** 
- **Lv3**: 新增 → **250**

#### 导弹炮台 (Missile)
- **Lv1**: 160 → **220** (+60, +37.5%)
- **Lv2**: 新增 → **260**
- **Lv3**: 新增 → **300**

#### 冰冻炮台 (Frost)
- **Lv1**: 120 → **170** (+50, +41.7%)
- **Lv2**: 新增 → **200**
- **Lv3**: 新增 → **240**

#### 闪电炮台 (Lightning)
- **Lv1**: 140 → **200** (+60, +42.9%)
- **Lv2**: 新增 → **240**
- **Lv3**: 新增 → **280**

### 2. 太空射击游戏 (spaceshooter)

#### 普通炮台 (Normal)
- **范围**: 80 → **150** (+70, +87.5%)

#### 爆发炮台 (Burst)
- **范围**: 120 → **180** (+60, +50%)

#### 其他炮台（已合理）
- **全屏炮台 (Wide)**: BASE_W * 2 = 720 (已是全屏)
- **狙击炮台 (Sniper)**: BASE_W * 0.8 = 288 (已足够大)

## 游戏体验改进

### 正面效果
✅ **更合理的布局空间**：玩家不需要过于密集地摆放炮台  
✅ **更好的策略性**：可以根据敌人路径灵活布置防御  
✅ **升级更有意义**：升级不仅提升伤害，还显著增加覆盖范围  
✅ **降低挫败感**：减少因范围太小而漏怪的情况  

### 平衡性考虑
- 炮台成本保持不变，性价比提升
- 敌人数量和强度未调整，整体难度曲线保持
- 高范围配合适当的射速和伤害，避免过度强大

## 技术说明

### Canvas坐标系
- 游戏画布尺寸：400x700（RPG塔防）或 360x640（太空射击）
- 攻击范围以像素为单位
- 范围值应该小于画布对角线长度的一半

### 范围计算示例
对于400x700的画布：
- 对角线长度 ≈ √(400² + 700²) ≈ 806像素
- 一半对角线 ≈ 403像素
- 最大合理范围：约300-350像素（留有余量）

当前配置的最大范围300像素是合理的，约占画布高度的43%。

## 后续优化建议

如果玩家反馈范围仍然不足，可以考虑：
1. 进一步增加基础范围10-20%
2. 添加"范围扩展"道具或技能
3. 允许某些特殊炮台拥有超大范围（如雷达塔）
4. 在UI中显示炮台范围圈，帮助玩家规划布局
