## kids-game-auto-test 项目代码优化

### 完成内容

对 `kids-game-auto-test` 自动化测试平台进行了全面的代码优化，版本从 1.0.0 升至 1.1.0。

#### 修复的架构 Bug
1. **日志分析器未绑定页面**：`LogAnalyzer.startCollecting(page)` 从未被调用，导致日志收集完全失效。在 `orchestrator.js` 中修复了调用时序——先 `launchBrowser()`，再 `startCollecting(page)`，再 `navigateToGame()`。
2. **性能监控重复导航**：`PerformanceMonitor.measureLoadTime()` 会对游戏页面重新 `page.goto()`，造成重复导航、测试状态丢失。修复为使用 Navigation Timing API 从已打开的 page 读取时间数据。

#### 新增文件
- `src/utils/helpers.js`：通用工具函数（`withRetry`、`sleep`、`takeScreenshot`、`formatDuration`、`formatBytes`、`groupBy` 等）

#### 重写/优化文件

- `src/game-simulator.js`：拆分 `launchBrowser()` 和 `runTests()`，添加失败自动截图，`runScenario()` 通用包装器，根据游戏类型动态执行额外测试场景
- `src/performance-monitor.js`：修复重复导航，从配置文件读取阈值，新增 LCP 指标，改进帧率采样算法（2秒时间窗口）
- `src/report-generator.js`：现代化 HTML 报告（性能进度条、通过率圆环图），Excel 增加 Performance 工作表，摘要计算缓存
- `src/log-analyzer.js`：URL 去重，HTTP 5xx/4xx 分级，明确 `startCollecting` 调用时序要求
- `src/ai-experience-analyzer.js`：修复 `analyze(page)` 参数不一致，延迟 Python 检测，支持页面截图输入
- `src/index.js`：新增 `--parallel/--timeout/--config` 参数，美化启动 Banner，优雅退出处理
