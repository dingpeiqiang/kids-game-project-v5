# 游客试玩功能实现说明

## 功能概述

实现了游客试玩功能，允许未注册用户体验游戏10分钟，时间到期后提示注册或登录。

## 实现细节

### 1. 核心文件修改

#### 1.1 认证工具 (`kids-game-frontend/src/utils/auth.ts`)
- 修改 `validateGameStartPermission()` 函数
- 添加游客模式检查逻辑
- 实现10分钟时间限制验证

#### 1.2 登录页面 (`kids-game-frontend/src/modules/login/index.vue`)
- 添加"游客试玩（10分钟）"按钮
- 实现 `startGuestMode()` 函数
- 设置游客标识和开始时间

#### 1.3 游戏页面 (`kids-game-frontend/src/modules/game/index.vue`)
- 添加游客时间跟踪逻辑
- 实现定时检查机制（每30秒检查一次）
- 时间到期自动跳转登录页

### 2. 技术实现

#### 2.1 数据存储
使用 localStorage 存储以下信息：
- `isGuest`: 标识是否为游客模式
- `guestStartTime`: 记录游客模式开始时间戳

#### 2.2 时间检查机制
- 在游戏页面启动时设置定时器
- 每秒更新一次剩余时间显示
- 实时倒计时提示用户剩余时间
- 超过10分钟自动提示并跳转

#### 2.3 权限验证
- 在游戏启动前验证游客身份
- 检查试玩时间是否已过期
- 过期则阻止游戏启动并引导登录

### 3. 用户体验流程

1. **进入登录页** → 看到"游客试玩"选项
2. **点击游客试玩** → 弹出确认对话框，显示10分钟限制说明
3. **确认后进入游戏** → 设置游客标识和开始时间
4. **游戏体验** → 页面顶部显示实时倒计时
5. **时间检查** → 后台每秒更新剩余时间
6. **时间到期** → 弹出提示框引导注册/登录
7. **自动跳转** → 跳转到登录页面

### 4. 安全考虑

- 时间检查在前端进行，防止简单绕过
- 使用 localStorage 存储状态，页面刷新后依然有效
- 时间计算基于服务器时间戳，相对准确

### 5. 扩展建议

1. **后端验证**: 可在后端添加游客会话管理
2. **时间同步**: 使用服务器时间而非客户端时间
3. **数据统计**: 记录游客转化率和行为数据
4. **个性化提示**: 根据游戏类型提供不同试玩时长

## 测试要点

1. 游客模式正常启动游戏
2. 10分钟后正确提示并跳转
3. 页面刷新后时间继续计算
4. 登录后清除游客状态
5. 不同浏览器间状态隔离

## 注意事项

- 当前实现为前端控制，高级用户可能绕过
- 建议后续增加后端验证机制
- 可根据业务需求调整试玩时长
