## 🎮 游戏特性

### 核心玩法
- 国风卡通风格的射击游戏
- 龙沿着预设或自定义路线移动
- 玩家控制角色射击龙的身体节
- 身体节独立血量系统
- 连击和分数系统

### 路线系统
- **关卡专属路线**：为每个关卡配置独立路线（新增）
- **预设路线**：4种预设路线（波浪、横冲、螺旋、之字）
- **自定义路线**：玩家可以绘制自己的路线
- **持久化存储**：
  - localStorage：浏览器本地存储
  - JSON文件：导出/导入到代码目录（新增）
- **动态速度**：根据路线长度自动计算合适的移动速度
- **关卡管理界面**：可视化管理所有关卡的路线配置

### 龙的系统
- **多种龙类型**：small, medium, large, elite, boss, treasure, coin
- **身体节独立血量**：每个身体节有独立的 HP
- **节点不重叠**：动态计算间距，确保身体节清晰分离
- **节点回退**：摧毁节点后，后续节点自动填补空缺
- **龙头无敌**：龙头不能被攻击

### 道具系统
- ⚔️ 攻击力提升
- 🔫 多重射击
- 💥 穿透增强
- ❤️ 生命回复

## 🔧 技术实现

### 路线跟随器 (RouteFollower)
- 基于路线点的插值移动
- 支持任意长度的路线
- 动态速度计算（目标18秒走完）
- 进度追踪和位置查询

### 身体节管理
- 每个身体节独立的位置计算
- 基于路线点索引的分布算法
- 平滑跟随动画
- 碰撞检测和伤害处理

### 持久化存储
- 使用 localStorage 保存自定义路线
- JSON 序列化/反序列化
- 错误处理和降级策略
- 删除功能支持

### 📝 关于代码拆分

曾尝试将代码拆分为多个模块，但发现：
1. **代码耦合度高**：2600+ 行代码中有大量相互依赖
2. **TypeScript 循环依赖**：类型定义和实现互相引用
3. **维护成本高**：拆分后需要大量导入/导出语句
4. **稳定性优先**：当前单文件运行稳定，无重大 bug

**决策**：保持当前单文件结构，专注于功能开发和优化。
