### 2. 模块化设计

每个模块职责清晰：

- **types.ts**: 定义所有 TypeScript 接口和类型
- **config.ts**: 集中管理游戏配置常量（难度、速度、颜色等）
- **cookie.ts**: 饼干的生成、更新和碰撞检测逻辑
- **particles.ts**: 粒子效果的创建和更新
- **renderer.ts**: Canvas 渲染相关的所有绘制函数
- **input.ts**: 鼠标和触摸事件的统一处理
- **gameState.ts**: 游戏状态管理、道具系统
- **index.ts**: 协调各模块，组织游戏主循环

## 🔧 使用示例

### 调整游戏难度

只需修改 `config.ts` 中的配置：

```typescript
export const GAME_CONFIG: GameConfig = {
  spawnInterval: 2000,           // 更慢的生成速度
  baseVerticalSpeed: 2,          // 更慢的上升速度
  maxCookies: 2,                 // 更少的同时存在饼干
}
```

### 添加新饼干类型

在 `config.ts` 中添加：

```typescript
export const COOKIES: CookieTemplate[] = [
  // ... 现有饼干
  { shape: 'diamond', emoji: '💎', color: '#00FFFF' },
]
```

### 添加新道具

1. 在 `config.ts` 添加图标
2. 在 `gameState.ts` 的 `usePowerup` 函数添加逻辑
