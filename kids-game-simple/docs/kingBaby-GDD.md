## §1 一句话与玩家动机

### 1.1 电梯演讲（≤ 30 字）

> Q版王者萌系1v1轻MOBA，一键开战，碎片化对局，轻松闯关乱斗

### 1.2 核心循环（约每 30 秒）

```
[操作] → [即时反馈] → [小目标达成] → [更大追求]
```

| 环节 | 本游戏具体是什么 |
|------|------------------|
| 操作 | 触控拖动英雄走位、点击释放技能、普攻自动攻击 |
| 即时反馈 | 技能命中特效、血量跳动、英雄弹跳动效、短促击杀音效 |
| 小目标 | 击杀敌方小兵、打掉对手血量、拿下单波对线优势 |
| 更大追求 | 摧毁敌方水晶、通关关卡、解锁新萌系英雄、刷新对局最高分 |

### 1.3 设计支柱（3 条，互不打架）

1. 轻量化魔改王者IP，Q版萌系画风，贴合儿童审美，合规无暴力内容
2. 砍掉复杂出装、铭文、排位系统，1分钟上手，碎片化超短对局适配Web小游戏
3. 操作极简、反馈饱满，轻度竞技+解压收割，兼顾可玩性与休闲属性

### 1.4 不做清单

- 不做5v5多人联机、复杂社交、段位排位、付费氪金系统
- 不做原版王者血腥技能特效、写实模型、军事对抗文案
- 不做长线肝度养成、随机抽奖、赌博类奖励机制（儿童合规）
- 不做高包体资源、复杂3D渲染，严控包体≤5MB适配安卓WebView

## §2 体验质量设计（必填 · 定稿前逐项打分 1～5）

> 儿童向小游戏也要 **爽感可描述、可验收**；本节是策划主战场，程序与美术按表实现对齐。

### 2.1 吸引力（首局 10 秒 + 回流）

| 维度 | 设计要求 | 本游戏方案 |
|------|----------|------------|
| 钩子 | 第一屏/第一波让玩家立刻「能动、能赢一点」 | 进入对局自动出兵，英雄初始满状态，3秒即可走位普攻，首波必击杀小兵获得金币奖励 |
| 节奏 | 前 30 秒信息密度：易学不空、不劝退 | 无长篇文字引导，图标可视化技能，仅保留走位+2个技能，30秒完成对线+首次击杀 |
| 主题辨识度 | 一句话能跟竞品区分开（龙/植物/球…） | Q版萌化王者英雄单人闯关MOBA，无复杂联机，纯休闲儿童向IP小游戏 |
| 回流理由 | 下次打开的动力：新关、纪录、收集、每日挑战… | 解锁新萌系英雄、闯关星级收集、单日最高分纪录、每日免费皮肤碎片奖励 |

### 2.2 玩法（深度与变化）

| 维度 | 设计要求 | 本游戏方案 |
|------|----------|------------|
| 主机制 | 玩家 80% 时间在重复的核心动作 | 英雄走位拉扯、技能释放清线、对战AI敌方英雄、推进小兵攻破水晶 |
| 变化源 | 关卡、敌人、道具、地形、规则突变（选 2+） | 关卡难度梯度、敌方AI英雄轮换、局内增益buff、草丛地形伏击机制 |
| 决策点 | 每局至少 N 次「选 A 还是 B」（含走位/建造/合成） | 每局5次决策：清小兵/消耗敌方、蓄力技能/瞬发技能、后撤回血/激进推进 |
| 难度曲线 | 失败原因可理解；难在技巧而非看不懂 | 前期AI迟钝易通关，中后期AI技能释放提速；失败原因：技能放空、贪输出被反打 |
| 单局时长 | 目标分钟数；何时该紧张、何时该喘息 | 单局2-3分钟；前期清线休闲喘息、后期攻水晶紧张团战，节奏松紧交替 |

### 2.3 竞技感（含单人「跟自己对战」）

| 维度 | 设计要求 | 本游戏方案 |
|------|----------|------------|
| 可量化成绩 | 分、波次、时间、连击、星级… | 对局得分、击杀数、通关时长、关卡1-3星评级、技能连击计数 |
| 比较对象 | 历史最高、本关目标、好友榜（若不做写「无」） | 个人历史最高分、关卡星级目标、无好友排行榜（儿童合规简化） |
| 公平感 | 输是因为操作/策略，不是 RNG 羞辱 | 无随机暴击、无随机buff，胜负完全由走位、技能释放操作决定，无恶意随机惩罚 |
| 高光时刻 | 可截图/可复述的「神操作」场景 | 残血反杀AI英雄、一波技能清屏小兵、极速1分钟通关关卡 |

### 2.4 解压感（标签含解压重点填）

| 维度 | 设计要求 | 本游戏方案 |
|------|----------|------------|
| 释放点 | 清屏、爆炸、连击断龙、收割…具体画面 | 大招全屏柔光粒子清小兵、敌方英雄软萌击退特效、水晶碎裂闪光特效 |
| 节奏 | 紧张—释放周期；BGM/密度是否配合 | 20秒对线拉扯紧张→5秒大招清怪释放；BGM同步从轻快转为激昂再回落 |
| 惩罚力度 | 失误可挽回 vs 硬死亡；儿童向偏温和 | 英雄倒地后5秒自动复活，无局内扣除奖励、无关卡重置惩罚，容错极高 |
| 重复劳动 | 机械操作是否有自动/辅助，避免手指累 | 普攻全自动释放，仅技能手动点击，支持一键自动清线，降低触控游学币 |

### 2.5 反馈（操作 → 感官）

| 类型 | 设计要求 | 本游戏方案（触发时机） |
|------|----------|------------------------|
| 视觉 | 受击闪白、缩放弹跳、飘字伤害/金币 | 英雄受击淡白柔光、命中目标轻微缩放、金币+1、伤害浅色飘字，无血腥数值 |
| 特效 | 粒子、拖尾、全屏轻微震屏、慢动作（慎用） | 技能柔和粒子拖尾、通关轻微弱震屏、无慢动作，特效低饱和度护眼 |
| 听觉 | 命中/击杀/升级/失败 分层音效，忌长时间同一音 | 普攻轻叮咚、击杀软萌音效、通关欢快短音、失败低沉舒缓音效，分段音频循环 |
| 触觉 | 震动（若平台支持）与重大事件绑定 | 安卓WebView适配：大招释放、通关触发弱手机震动，普攻无震动 |
| UI 反馈 | 按钮态、进度条、波次预告、危险提示色 | 技能按钮按压变色、血量进度条渐变、敌方高亮红色危险提示 |

**反馈强度阶梯（建议填表）**：

| 事件级别 | 示例 | 视 | 听 | 特效 |
|----------|------|----|----|------|
| 轻 | 普攻命中 | 小幅伤害飘字 | 短促轻叮咚 | 微小光点粒子 |
| 中 | 击杀小兵、技能命中英雄 | 金币弹窗、目标弹跳 | 软糯击杀音效 | 圈层柔光特效 |
| 重 | BOSS英雄击杀、通关、濒死 | 全屏柔光提亮、星级弹出 | 欢快合奏音效 | 大范围花瓣粒子、弱震屏 |

### 2.6 奖励与成长

| 维度 | 设计要求 | 本游戏方案 |
|------|----------|------------|
| 即时奖励 | 每 20～40 秒一次「拿到东西」的感觉 | 每击杀小兵掉落金币，30秒获取局内buff道具，即时弹窗奖励提示 |
| 局内成长 | 升级、选 Buff、资源变强… | 自动等级升级、随机拾取护盾/增伤温和buff，无复杂装备选择 |
| 局外成长 | 解锁关、皮肤、图鉴（若无写「纯局内」） | 闯关解锁新Q版英雄、解锁纯色萌系皮肤、英雄图鉴收集 |
| 随机奖励 | 宝箱、暴击…概率与保底（儿童向避免赌博表述） | 通关固定奖励+额外福利礼包，无概率抽奖，保底解锁皮肤碎片 |
| 失败补偿 | 失败仍保留部分进度或明确「学到了什么」 | 失败保留本局金币进度，弹窗提示操作优化技巧，无需重开关卡 |

### 2.7 画风与可读性

| 维度 | 设计要求 | 本游戏方案 |
|------|----------|------------|
| 整体调性 | 可爱/Q 版/扁平/低多边形…与受众一致 | 2D扁平Q版萌系、马卡龙低饱和配色、软轮廓卡通画风 |
| 剪影与对比 | 小屏上角色、子弹、障碍一眼可分 | 英雄粗轮廓描边、技能色块区分、草丛障碍高对比度，手机小屏清晰识别 |
| 色彩语义 | 友方/敌方/危险/可交互 与 §4 色板一致 | 友方浅青、敌方浅红、危险区域橙黄、技能按钮马卡龙亮色，绑定色板 |
| 动效风格 | 弹跳、果冻、硬直…统一物理感 | 全角色果冻弹跳动效、受击软回弹，无僵硬硬直动作 |
| 禁忌 | 血腥恐怖写实枪炮等（儿童合规） | 删除原版刀刃、血迹、写实盔甲，全部萌化柔化，零暴力元素 |

### 2.8 体验验收清单（策划自测）

- [√] 新手不读长文能在 1 分钟内开始玩
- [√] 连续玩 3 局不腻（第2局新buff、第3局新敌方英雄）
- [√] 赢一局有明确「爽点」；输一局知道怎么改操作
- [√] 音效+特效不糊成一团（重事件才上大特效）
- [√] 与 §1 设计支柱无冲突

## §4 规则与内容表（策划数值 · 程序对照 `config`）

> 改平衡只改本节表格，避免口头改需求。

### 4.1 胜负与评分

| 条件 | 规则 |
|------|------|
| 胜利 | 摧毁敌方萌版水晶，即可判定本局胜利 |
| 失败 | 己方水晶血量归零、对局超时4分钟，判定失败 |
| 评分 / 星级 | 3星：1.5分钟内通关；2星：2.5分钟内通关；1星：正常通关；超时仅1星 |

| 操作 | 玩家看到的说明 |

| 大招按钮 | 释放全屏大招，清理全部小兵 |
| 自动作战 | 开启后自动走位、自动释放普攻 |
