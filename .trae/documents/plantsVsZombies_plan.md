# 植物大战僵尸高仿版 - 策划文档

## 一、游戏概述

### 1.1 游戏定位
《植物大战僵尸》是一款经典的塔防类游戏，玩家通过种植各种植物来防御一波波来袭的僵尸，保护自己的家园。本项目旨在打造一个网页端的高仿版本，保留核心玩法的同时优化用户体验。

### 1.2 核心玩法
- **种植系统**：玩家点击草地格子种植植物，消耗阳光资源
- **资源系统**：向日葵生产阳光，是种植植物的基础资源
- **防御系统**：各种植物拥有不同攻击方式（远程、近战、辅助等）
- **僵尸波次**：僵尸从右侧入场，沿道路行进，目标是吃掉左侧的大脑
- **关卡系统**：多个关卡，难度递增，解锁新植物和僵尸类型

### 1.3 游戏特色
- 5行9列的经典战斗场地布局
- 10+种植物，各有独特能力
- 8+种僵尸类型，各具特点
- 阳光资源管理策略
- 策略性防御布局

## 二、技术架构设计

### 2.1 项目结构（遵循游戏开发规范）

```
games/plantsVsZombies/
├── index.ts          # 游戏入口，暴露 initPlantsVsZombies()
├── game.ts           # 主游戏类，游戏循环
├── config.ts         # 所有可配置常量（植物、僵尸、关卡配置）
├── types.ts          # 所有类型/接口定义
├── logic/            # 纯逻辑模块
│   ├── plants.ts     # 植物逻辑
│   ├── zombies.ts    # 僵尸逻辑
│   ├── projectiles.ts # 子弹/投射物逻辑
│   └── waveManager.ts # 波次管理
└── render/           # 纯渲染模块
    ├── background.ts # 背景渲染
    ├── plants.ts     # 植物渲染
    ├── zombies.ts    # 僵尸渲染
    ├── ui.ts         # UI渲染
    └── effects.ts    # 特效渲染
```

### 2.2 分层架构
| 层级 | 职责 | 文件 |
|------|------|------|
| **编排层** | 游戏循环、事件绑定、模块调用 | `game.ts` |
| **逻辑层** | 植物行为、僵尸AI、碰撞检测、波次控制 | `logic/` |
| **渲染层** | 画布绘制、动画效果、UI显示 | `render/` |
| **配置层** | 植物属性、僵尸属性、关卡数据 | `config.ts` |
| **类型层** | TypeScript接口定义 | `types.ts` |

#### 3.1.2 植物类型
```typescript
interface Plant {
  id: string;
  name: string;
  sunCost: number;        // 阳光消耗

  lastAttackTime: number;

type PlantType = 
  | 'sunflower'      // 向日葵 - 生产阳光
  | 'peashooter'     // 豌豆射手 - 直线射击
  | 'sunflower_double' // 双子向日葵
  | 'snowpea'        // 寒冰射手 - 减速
  | 'repeater'       // 双重射手
  | 'wallnut'        // 坚果墙 - 防御
  | 'cherry_bomb'    // 樱桃炸弹 - 范围爆炸
  | 'potato_mine'    // 土豆地雷
  | 'chomper'        // 大嘴花 - 吞噬
  | 'sunshroom'      // 阳光蘑菇
```

## 四、游戏流程设计

### 4.1 游戏主循环

```
开始游戏 → 初始化关卡 → 游戏循环 → 游戏结束
              ↓              ↓
        加载初始阳光      更新所有实体
              ↓              ↓
        显示植物卡片      检测碰撞
              ↓              ↓
        等待玩家操作      处理游戏事件
```

### 4.2 游戏循环流程

1. **输入处理**：检测点击/触摸事件
2. **阳光生产**：向日葵自动生产阳光
3. **僵尸生成**：按波次配置生成僵尸
4. **植物攻击**：植物自动攻击范围内的僵尸
5. **僵尸移动**：僵尸沿道路前进
6. **碰撞检测**：子弹击中僵尸、僵尸到达终点、僵尸啃食植物
7. **状态更新**：更新生命值、移除死亡实体
8. **渲染画面**：绘制所有游戏元素

### 4.3 胜利/失败条件

- **胜利条件**：成功抵御所有波次的僵尸攻击
- **失败条件**：僵尸到达左侧终点（吃掉大脑）或生命值归零

## 七、技术要点

### 7.1 渲染优化
- 使用requestAnimationFrame驱动渲染
- 精灵图优化绘制性能
- 离屏Canvas绘制复杂场景

### 7.2 性能考虑
- 僵尸数量限制（同屏不超过20个）
- 对象池复用僵尸和投射物
- 碰撞检测优化（只检测可见区域）

### 7.3 移动端适配
- 触摸事件支持
- 响应式布局
- 简化操作界面

### 7.4 代码规范
- 遵循项目已有的TypeScript规范
- 逻辑与渲染严格分离
- 配置数据集中管理

## 十、风险与应对

### 10.1 潜在风险

| 风险 | 描述 | 应对措施 |
|------|------|---------|
| 性能问题 | 大量僵尸和投射物导致卡顿 | 对象池复用、碰撞检测优化 |
| 内存泄漏 | 僵尸/植物移除不彻底 | 定期清理死亡实体 |
| 移动端适配 | 触摸操作不流畅 | 优化触摸事件处理 |
| 代码复杂度 | 逻辑与渲染耦合 | 严格分层架构 |

### 10.2 测试计划

- **单元测试**：植物攻击、僵尸移动、碰撞检测等核心逻辑
- **集成测试**：完整游戏流程测试
- **性能测试**：大量僵尸场景下的帧率测试
- **兼容性测试**：不同浏览器和设备适配测试
