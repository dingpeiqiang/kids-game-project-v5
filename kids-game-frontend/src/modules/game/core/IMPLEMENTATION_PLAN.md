# 游戏模式架构 - 分阶段实现计划

## 📋 总体规划

基于架构分析和优化建议，我们将分 **5 个阶段** 逐步实现这套架构。

---

## 🎯 阶段划分

```
阶段 1: 单人模式基础架构
    ↓
阶段 2: 本地对抗模式
    ↓
阶段 3: 组队模式
    ↓
阶段 4: 多人模式
    ↓
阶段 5: 网络对抗模式
```

---

## 📦 阶段 1: 单人模式基础架构

### 目标
- 实现统一模式管理器基础框架
- 实现单人模式（1个游戏实例 + 1个玩家）
- 实现基础的数据收集和规则判定

### 核心任务

#### 1.1 创建统一模式管理器
```
文件: UnifiedModeManager.ts

功能:
├── 模式类型枚举
├── 游戏实例配置接口
├── 规则配置接口
├── 根据模式创建游戏实例的方法
├── 游戏实例生命周期管理
└── 数据收集和规则判定基础框架
```

#### 1.2 实现单人模式策略
```
功能:
├── 创建1个游戏实例
├── 添加1个玩家
├── 监听游戏事件（state_change, game_end）
├── 收集游戏数据
└── 应用单人模式规则（AI难度、时间限制）
```

#### 1.3 游戏实例管理
```
功能:
├── 创建游戏实例
├── 启动游戏实例
├── 暂停/恢复游戏
├── 重启游戏
└── 销毁游戏实例
```

#### 1.4 数据收集
```
功能:
├── 监听游戏事件
├── 收集游戏状态
├── 存储玩家数据
└── 统计信息
```

### 交付物

- ✅ `UnifiedModeManager.ts` - 统一模式管理器
- ✅ `GameInstance.ts` - 游戏实例类型定义
- ✅ `SinglePlayerStrategy.ts` - 单人模式策略（可选，可内联到管理器）
- ✅ 单元测试：测试单人模式创建和管理
- ✅ 使用示例：展示如何使用单人模式

### 验收标准

- [ ] 可以成功创建单人模式
- [ ] 可以正确收集游戏数据
- [ ] 可以正确应用单人模式规则
- [ ] 游戏可以正常启动、暂停、恢复、停止
- [ ] 单元测试通过率 ≥ 90%

## 🎮 阶段 2: 本地对抗模式

### 目标
- 实现本地对抗模式（2个游戏实例 + 物理分屏）
- 实现分屏布局管理
- 实现双实例数据收集和规则判定

### 核心任务

#### 2.1 实现本地对抗策略
```
功能:
├── 创建2个游戏实例
├── 添加2个玩家（各自独立）
├── 设置物理分屏（左/右或上/下）
├── 监听两个游戏实例的事件
├── 收集两个游戏的数据
└── 应用对抗模式规则（胜负判定）
```

#### 2.2 分屏布局管理
```
功能:
├── 创建两个独立容器
├── 设置容器布局（垂直/水平分屏）
├── 添加玩家标识
├── 响应式布局适配
└── 容器清理
```

#### 2.3 双实例数据收集
```
功能:
├── 监听两个游戏实例的事件
├── 收集两个游戏的状态
├── 对比两个游戏的状态
└── 实时状态同步
```

#### 2.4 对抗规则判定
```
功能:
├── 判定胜负（分数对比、生命值对比）
├── 处理平局情况
├── 触发结束事件
└── 输出对战结果
```

### 交付物

- ✅ `LocalBattleStrategy.ts` - 本地对抗策略
- ✅ `SplitScreenLayout.ts` - 分屏布局管理
- ✅ 单元测试：测试双实例创建和管理
- ✅ 集成测试：测试完整的本地对抗流程
- ✅ 使用示例：展示如何使用本地对抗模式

### 验收标准

- [ ] 可以成功创建本地对抗模式
- [ ] 分屏布局正确显示
- [ ] 两个游戏实例完全隔离
- [ ] 可以正确收集两个游戏的数据
- [ ] 可以正确判定胜负
- [ ] 单元测试通过率 ≥ 90%
- [ ] 集成测试通过率 ≥ 80%

## 👥 阶段 3: 组队模式

### 目标
- 实现组队模式（1个游戏实例 + 多个玩家，同一队伍）
- 实现团队计分策略
- 实现团队协作规则

### 核心任务

#### 3.1 实现组队策略
```
功能:
├── 创建1个游戏实例
├── 添加多个玩家（同一队伍）
├── 设置团队标识
├── 监听游戏事件
├── 收集所有玩家的数据
└── 应用组队模式规则
```

#### 3.2 团队计分策略
```
策略:
├── sum: 团队总分 = 所有玩家分数之和
├── average: 团队总分 = 所有玩家分数的平均值
└── max: 团队总分 = 所有玩家分数的最大值
```

#### 3.3 团队协作规则
```
规则:
├── 团队共享生命值（可选）
├── 团队共享时间限制（可选）
├── 玩家之间互相帮助（可选）
└── 团队胜利条件
```

### 交付物

- ✅ `TeamStrategy.ts` - 组队策略
- ✅ `TeamScoringService.ts` - 团队计分服务
- ✅ 单元测试：测试团队计分策略
- ✅ 使用示例：展示如何使用组队模式

### 验收标准

- [ ] 可以成功创建组队模式
- [ ] 可以正确添加多个玩家
- [ ] 团队计分策略正确
- [ ] 可以正确应用团队协作规则
- [ ] 单元测试通过率 ≥ 90%

## 🌐 阶段 5: 网络对抗模式

### 目标
- 实现网络对抗模式（2个游戏实例 + 网络同步）
- 实现网络通信
- 实现状态同步
- 实现延迟补偿
- 实现防作弊

### 核心任务

#### 5.1 实现网络对抗策略
```
功能:
├── 创建2个游戏实例（每个设备1个）
├── 建立 WebSocket 连接
├── 房间匹配
├── 玩家身份验证
├── 监听游戏事件
├── 收集两个游戏的数据
└── 应用网络对抗规则
```

#### 5.2 网络通信
```
功能:
├── 建立 WebSocket 连接
├── 发送游戏数据
├── 接收游戏数据
├── 断线重连
└── 心跳检测
```

#### 5.3 状态同步
```
功能:
├── 同步游戏状态
├── 处理网络延迟
├── 状态一致性检查
└── 冲突解决
```

#### 5.4 延迟补偿
```
技术:
├── 客户端预测
├── 服务器权威
├── 插值和外推
└── 重新执行
```

#### 5.5 防作弊
```
机制:
├── 服务器验证
├── 数据加密
├── 异常检测
└── 封禁机制
```

### 交付物

- ✅ `OnlineBattleStrategy.ts` - 网络对抗策略
- ✅ `NetworkService.ts` - 网络服务
- ✅ `StateSynchronizer.ts` - 状态同步器
- ✅ `AntiCheatService.ts` - 防作弊服务（可选）
- ✅ 单元测试：测试网络通信和状态同步
- ✅ 集成测试：测试完整的网络对抗流程
- ✅ 使用示例：展示如何使用网络对抗模式

### 验收标准

- [ ] 可以成功创建网络对抗模式

- [ ] 基本的防作弊机制有效
